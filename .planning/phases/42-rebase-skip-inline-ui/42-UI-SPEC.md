## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (this phase) | "Skip Commit" |
| Secondary CTAs (existing) | "Continue Rebase", "Abort Rebase" |
| Empty state heading | Not applicable (button only appears during active rebase) |
| Empty state body | Not applicable |
| Error state (skip failure) | Toast: `{error.message}` or fallback "Skip failed" |
| Success state | Silent -- no toast (graph refresh + banner progress update provide feedback) |
| Destructive confirmation | Not applicable -- Skip is non-destructive forward action, no confirmation needed |

### Button label rationale
- "Skip Commit" rather than "Skip" -- explicitly communicates what is being skipped (the current conflicting commit)
- OperationBanner uses short "Skip" label because banner context already implies commit-level action
- StagingPanel uses longer "Skip Commit" because the inline form context is less obvious

## Layout Contract

### Button row (3 buttons)

```
[    Continue Rebase    ] [ Skip Commit ] [  Abort Rebase  ]
         flex: 3              flex: 1          flex: 2
```

- Container: `display: flex; gap: 6px;`
- All buttons share: `height: 34px; border-radius: 4px; font-size: 12px; font-weight: 600;`
- Flex ratios encode visual priority: Continue is the most prominent (3), Abort is secondary (2), Skip is tertiary (1)
