**`build_partial_patch_text`** (private helper):
```rust
fn build_partial_patch_text(
    file_path: &str,
    patch: &git2::Patch<'_>,
    hunk_idx: usize,
    selected_indices: &[u32],
) -> Result<String, TrunkError> {
```
Algorithm:
1. Collect `selected_indices` into a `HashSet<u32>` for O(1) lookup.
2. Get hunk via `patch.hunk(hunk_idx)?` -- extract `old_start`, `new_start`.
3. Iterate `patch.num_lines_in_hunk(hunk_idx)?` lines using `patch.line_in_hunk(hunk_idx, line_idx)?`.
4. For each line, get `line.origin()` (char: `+`, `-`, or ` `) and `line.content()` (bytes).
5. Convert content to string with `String::from_utf8_lossy(line.content())`.
6. Ensure content ends with `\n`. If not (EOF without newline), append `\n` and after the line, add `\ No newline at end of file\n`.
7. Build patch lines:
   - Origin `+` (add): If index is in selected_set, keep as `+{content}` and increment new_count. Otherwise SKIP entirely.
   - Origin `-` (delete): If index is in selected_set, keep as `-{content}` and increment old_count. Otherwise convert to context: ` {content}` and increment BOTH old_count and new_count.
   - Origin ` ` (context): Keep as ` {content}` and increment both old_count and new_count.
8. Check delta status for diff header: `patch.delta().status()`. If `git2::Delta::Added`, use `--- /dev/null`. If `git2::Delta::Deleted`, use `+++ /dev/null`. Otherwise use standard `--- a/{path}` / `+++ b/{path}`.
9. Format final patch:
```
diff --git a/{file_path} b/{file_path}
--- a/{file_path}       (or --- /dev/null)
+++ b/{file_path}       (or +++ /dev/null)
@@ -{old_start},{old_count} +{new_start},{new_count} @@
{patch_lines joined}
```
10. Return the patch text string.
