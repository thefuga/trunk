### New tokens (Phase 34 additions to app.css)

| Token | Value | Usage |
|-------|-------|-------|
| `--color-diff-add-bg-selected` | rgba(74, 222, 128, 0.25) | Background of selected add lines (brighter than unselected 0.1 alpha) |

## Interaction Contract

### Line selection

| Behavior | Specification |
|----------|--------------|
| Click on add/delete line | Toggle that line in/out of the selection set |
| Shift+click on add/delete line | Range-select all add/delete lines between last-clicked and shift-clicked line (skipping context lines) |
| Click context line | No-op. Context lines are not selectable. |
| Click line in different hunk | Clear existing selection, start new selection in the clicked hunk |
| Escape key | Clear all selected lines, restore hunk-mode toolbar buttons |
| Navigate to different file | Clear all selected lines |
| After stage/unstage/discard lines completes | Clear all selected lines (diff re-fetches) |
| Commit diff mode | No selection behavior. Lines remain read-only. |

### Cursor contract

| Line origin | Diff kind (unstaged/staged) | Cursor |
|-------------|-----------------------------|--------|
| Add | unstaged or staged | `pointer` |
| Delete | unstaged or staged | `pointer` |
| Context | any | `default` |
| Any | commit | `default` |

### Toolbar mode switching

The hunk toolbar row has two mutually exclusive states:

**Hunk mode (no lines selected in this hunk):**

| Diff kind | Buttons displayed (left to right) |
|-----------|----------------------------------|
| unstaged | "Discard Hunk" (destructive style) + "Stage Hunk" (stage style) |
| staged | "Unstage Hunk" (unstage style) |
| commit | (no buttons) |

**Selection mode (lines selected in this hunk):**

| Diff kind | Buttons displayed (left to right) |
|-----------|----------------------------------|
| unstaged | "Discard Lines (N)" (destructive style) + "Stage Lines (N)" (stage style) |
| staged | "Unstage Lines (N)" (unstage style) |
| commit | (not reachable -- selection disabled) |

Where N is the count of selected lines. Buttons replace, not supplement, the hunk-mode buttons. Only one mode visible per toolbar at a time.

All buttons are disabled (opacity 0.4, cursor not-allowed) when `hunkOperationInFlight` is true.

### Selected line visual

| State | Background | Text color |
|-------|------------|------------|
| Add line, unselected | `var(--color-diff-add-bg)` | `var(--color-diff-add)` |
| Add line, selected | `var(--color-diff-add-bg-selected)` | `var(--color-diff-add)` |
| Delete line, unselected | `var(--color-diff-delete-bg)` | `var(--color-diff-delete)` |
| Delete line, selected | `var(--color-diff-delete-bg-selected)` | `var(--color-diff-delete)` |
| Context line | transparent | `var(--color-text)` |

Text color does not change between selected and unselected. Only the background changes.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (unstaged, hunk mode) | "Stage Hunk" |
| Primary CTA (unstaged, selection mode) | "Stage Lines (N)" |
| Primary CTA (staged, hunk mode) | "Unstage Hunk" |
| Primary CTA (staged, selection mode) | "Unstage Lines (N)" |
| Destructive action (hunk mode) | "Discard Hunk" |
| Destructive action (selection mode) | "Discard Lines (N)" |
| Discard lines confirmation title | "Discard Lines" |
| Discard lines confirmation body | "Discard N selected lines? This cannot be undone." |
| Stage lines success toast | (no toast -- silent success, diff refreshes) |
| Unstage lines success toast | (no toast -- silent success, diff refreshes) |
| Discard lines success toast | "Discarded N lines" |
| Error toast (stage lines) | "{error.message}" fallback "Stage lines failed" |
| Error toast (unstage lines) | "{error.message}" fallback "Unstage lines failed" |
| Error toast (discard lines) | "{error.message}" fallback "Discard lines failed" |
| Empty state | "Select a file or commit to view its diff" (existing, unchanged) |

Notes on copy decisions:
- "Lines" not "line" even for N=1 -- keeps the label stable and avoids singular/plural branching.
- Stage and unstage succeed silently (matching existing Stage Hunk / Unstage Hunk behavior). Only discard shows a success toast because it is destructive and irreversible.
- N is always an integer rendered directly: "Stage Lines (3)", not "Stage Lines (three)".
