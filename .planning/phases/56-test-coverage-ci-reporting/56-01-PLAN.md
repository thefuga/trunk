   ```bash
   bun add -D @vitest/coverage-v8
   ```

2. Add coverage configuration to the `test` section in vite.config.ts. The existing test block is:
   ```typescript
   test: {
     include: ["src/**/*.test.ts"],
     environment: "jsdom",
     setupFiles: ["./vitest-setup.ts"],
   },
   ```
   Add a `coverage` property so it becomes:
   ```typescript
   test: {
     include: ["src/**/*.test.ts"],
     environment: "jsdom",
     setupFiles: ["./vitest-setup.ts"],
     coverage: {
       provider: "v8",
       reporter: ["text", "lcov", "html"],
       reportsDirectory: "./coverage",
       include: ["src/**/*.ts", "src/**/*.svelte"],
       exclude: ["src/**/*.test.ts"],
     },
   },
   ```

   Key points per D-02 and research:
   - provider: "v8" -- uses V8's built-in coverage (fast, zero-config default)
   - reporter: ["text", "lcov", "html"] -- text for CI console output, lcov for PR comment action, html for artifact upload
   - reportsDirectory: "./coverage" -- standard output directory
   - include: covers all .ts and .svelte source files
   - exclude: prevents test files from inflating coverage numbers
   - NO thresholds (per D-04 -- report-only, no enforcement)

3. Verify tests still pass with coverage config present:
   ```bash
   bun run test
   ```

4. Verify coverage generation works locally:
   ```bash
   bun run test -- --coverage.enabled
   ```
   This should produce a `coverage/` directory with lcov.info, HTML report, and text summary in console.

