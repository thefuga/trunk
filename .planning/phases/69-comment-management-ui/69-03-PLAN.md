<objective>
Add the two read commands the panel needs to render the full review (CMT-01) and to flag stale anchors (CMT-04): `list_session_comments` (read-only clone of the session's comments) and `resolve_session_comments` (a git2-backed eager resolvability check, D-06). The resolver returns one `CommentResolution { id, resolvable, reason }` per comment, classifying unresolvable anchors as `CommitGone` / `FileGone` / `LineOutOfRange` (D-08) — never dropping an input, never panicking.

Purpose: The panel shows orphan badges up-front (the success criterion's "read-only orphaned state with a reason badge" without requiring a click). A frontend-only "is the oid in session.commits" check can't catch a renamed/deleted file or an out-of-bounds range — this MUST be git2-backed.
Output: Two new read commands registered in lib.rs, with a pure `resolve_all` classifier unit-tested against an in-process git2 repo for every orphan condition and the side-aware line-bound path.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frontend → IPC read command | Untrusted path + stored anchor metadata (commit_oid, file_path, line range) drive git2 object-DB lookups |
| session store → git2 | Persisted anchor fields read against the live repository object database |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-69-09 | Tampering / EoP | path traversal via `file_path` into the filesystem | mitigate | `file_path` is only ever passed to `Tree::get_path` (a git object-DB tree lookup), never to `std::fs`; it cannot escape the repo object DB; the session filename remains the FNV-1a hash of the canonical path (already tested) |
| T-69-10 | Denial of Service | malformed `commit_oid` crashing the resolver | mitigate | parse via `git2::Oid::from_str`; on failure classify `CommitGone` and continue — never panic, never bubble (mirrors intersect_graph_order's never-drop fallback) |
| T-69-11 | Denial of Service | a very large blob inflating line-count work | accept | bounded by the user's own repository (local, single-user); reading blob content is the same cost git already pays; no amplification |
| T-69-12 | Information Disclosure | resolver leaking content beyond resolvability | mitigate | resolve_all returns only { id, resolvable, reason } — no file content or line text crosses the boundary; excerpts come from the independently-stored cached_excerpt, not this command |
</threat_model>
