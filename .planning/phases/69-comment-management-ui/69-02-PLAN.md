## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-69-05 | Tampering | edit/delete targeting the wrong comment under a stale index | mitigate | target by stable uuid `id` (D-03), never by list position; a second tab reordering the list cannot misfire; missing id → not_found (edit) / no-op (delete) |
| T-69-06 | Spoofing | id collision letting two comments share an id | mitigate | uuid v4 generation (collision-free); never derived from content |
| T-69-07 | Denial of Service | unbounded comment text from the renderer | accept | local single-user desktop app; text is stored as-is in a local JSON the same user controls; no network amplification, no injection sink (text is data, not executed) |
| T-69-08 | Tampering | concurrent multi-tab writes losing data | mitigate | all three commands route through mutate_session_rmw (mutex held across read→mutate→save) — the proven 50-thread-tested primitive |
</threat_model>
