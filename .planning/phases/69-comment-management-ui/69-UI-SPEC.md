# Phase 69 — UI Design Contract

> Visual and interaction contract for the Comment Management UI: the real review panel that replaces the Phase 65 throwaway stub. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This is a **brownfield** phase in a dense desktop Git GUI (Tauri 2 + Svelte 5). The contract describes the EXISTING theme system (`src/app.css` CSS custom properties), narrowed to what this phase touches. It does NOT introduce a new design system, shadcn, or marketing-scale spacing — those would contradict the shipping app. Values below are extracted from `src/app.css` and the conventions in sibling panels (`CommitDetail.svelte`, `DiffPanel.svelte`, the `ReviewPanel.svelte` stub).

## Spacing Scale

Declared tokens for this phase (standard set only):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Inline button padding, tight gaps (e.g. icon ↔ label), button group gap |
| sm | 8px | Default gap between rows; gap between elements within a row |

Exceptions: none — only standard-set values are declared as tokens.

**Density guidance (not a declared scale — prose for the executor):** This panel must visually match the density of its sibling components in the same pane. For sub-token spacing the contract does NOT enumerate (e.g. the tight vertical padding inside a comment/commit list row, and the control border-radius), **match the existing `CommitDetail.svelte` / `DiffPanel.svelte` / `ReviewPanel.svelte` stub** rather than widening to the standard scale. Do NOT pad rows out to 16/24/32 — that would break visual consistency with the rest of the center pane. Larger tokens (16/24/32/48/64) are intentionally not used in this panel.

## Typography

Panel-class components in this app use 11–13px. Declared subset for this phase (3 roles):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Group header (commit SHA + summary) | 13px | 600 | 1.3 |
| Body (comment text, "Add note" label, button labels) | 12px | 400 | 1.5 |
| Secondary (orphan reason badge, line-range label, cached-excerpt caption) | 11px | 400 | 1.4 |

Weights: exactly two — **400** (default) and **600** (emphasis: group headers, the mono short SHA). Do not introduce 500/700.

Mono: short SHAs and the `path:Lstart-Lend` line-range label use `--font-mono` at the same size as their surrounding role.

## Color

All values are existing tokens from `src/app.css`. Do not introduce new hex values; if a new semantic is genuinely needed, add a `--color-*` token to `app.css` first.

| Role | Value (token) | Usage |
|------|---------------|-------|
| Dominant (60%) | `--color-surface` (#161b22) | Panel background |
| Secondary (30%) | `--color-bg` (#0d1117) / `--color-muted-bg` | Page behind panel; muted group-header / orphan badge backing |
| Accent (10%) | `--color-accent` (#388bfd) | See reserved-for list below |
| Destructive | `--color-danger` (#f87171) + `--color-danger-bg` + `--color-danger-border` | Delete affordance + delete confirmation |

Accent reserved for (explicit — NOT "all interactive elements"):
- The persistent **"Review" pane toggle** in its active/selected state (the way back from a jumped-to diff to the panel, per D-07).
- The **jump-to-anchor affordance** on a *resolvable* comment (hover/focus emphasis only).

Accent is NOT used for: selected commit rows (those already use `--color-selected-row`), comment row backgrounds, the "Add note" affordance, or edit controls. Those use `--color-text` / `--color-text-muted` / `--color-hover`.

Additional semantic tokens this phase consumes (already defined):
- **Orphaned comment** read-only state (D-08): reason badge uses `--color-warning` text on `--color-warning-bg`; the orphaned row body text is dimmed via `--opacity-dimmed` (0.4) on the anchor metadata only — the comment text and cached excerpt stay at full `--color-text` (text is stored independent of resolvability, Phase 65 D-04).
- **Text hierarchy:** primary `--color-text`; muted (timestamps-equivalent metadata, line-range labels, "no comments" empty copy) `--color-text-muted`.
- **Hover:** list rows and the "Add note" affordance use `--color-hover` on hover.

## Visual Hierarchy

**Focal point:** the **commit group header** (short SHA in `--font-mono` weight 600 + commit summary) is the per-group eye anchor. It is the largest type role (13px) and the heaviest weight (600) in the panel, and it opens each group — the reader's eye lands on it first and uses it to scan between commits. Comment rows beneath it are visually subordinate (12px/400 body). Nothing else in the panel competes for the focal role: the "Add note" affordance, edit/delete controls, and jump affordance are all secondary and only gain emphasis (accent / hover) on interaction.

## Copywriting Contract

These strings are net-new (no existing equivalents in the codebase) and are LOCKED here for the executor. The recipient framing of the milestone is an AI coding agent, but these strings are the human-facing panel UI.

| Element | Copy |
|---------|------|
| Primary CTA (per-commit, D-02) | **"Add note"** (icon: `MessageSquarePlus` from `@lucide/svelte`, with visible text label at 12px) |
| Panel empty state — no commits in session | **"No commits in this review yet."** + muted sub-line: **"Add commits from the graph to start reviewing."** |
| Panel empty state — commits present, no comments | **"No comments yet."** + muted sub-line: **"Select lines in a diff to comment, or add a note to a commit above."** |
| Commit with no comments (per-group) | muted inline: **"No comments on this commit."** |
| Edit affordance (inline, D-10) | trigger label **"Edit"**; in-place actions **"Save"** and **"Cancel"** |
| Empty-text validation | Save is disabled when the textarea is empty/whitespace-only (no error toast; the disabled Save is the feedback) |
| Error state (load/resolve failure) | toast via existing `showToast(..., "error")`: **"Failed to load review comments. Reload the panel to retry."** (mirrors the stub's `safeInvoke` + toast pattern) |
| Destructive confirmation (delete, D-05) | `@tauri-apps/plugin-dialog` `ask` body: **"Delete this comment? This cannot be undone."** title: **"Delete comment"** — cancel is a no-op |

Orphan reason badge labels (map the Rust `OrphanReason` enum → human strings, LOCKED):

| `OrphanReason` variant | Badge label |
|------------------------|-------------|
| `CommitGone` | **"commit gone"** |
| `FileGone` | **"file gone"** |
| `LineOutOfRange` | **"line out of range"** |

Jump affordance: icon-only `CornerDownRight` (or `ArrowRight`) from `@lucide/svelte` with `aria-label="Jump to code"`. Disabled (no pointer, dimmed, no aria action) on orphaned comments per D-08.

## Interaction Contract (phase-specific)

This section is non-template but load-bearing for the executor — the success criteria are interaction-defined.

| Interaction | Contract |
|-------------|----------|
| View comments (CMT-01) | Grouped by commit (D-09): a group header per commit (short SHA `--font-mono` 600 + summary), comments nested beneath. Order within group follows backend list order. |
| Add commit-level note (ANCH-03 / D-02) | "Add note" affordance per commit row → opens the same inline textarea used for edit; Save writes a commit-level comment (no anchor). |
| Edit (CMT-02 / D-10) | Inline: clicking "Edit" swaps the comment text for a textarea in place with Save/Cancel. Works identically for line-anchored, commit-level, and orphaned comments. No navigation, no separate composer. |
| Delete (CMT-03 / D-05) | "Delete" → native `ask` confirm (copy above) → on confirm, `delete_comment` by `id`. Cancel writes nothing. |
| Jump (CMT-04 / D-07) | Resolvable comment only: selects commit + file, swaps the pane from panel → diff/full-file per `Source`, scrolls to and highlights the line range. A persistent "Review" toggle (accent when active) returns to the panel. |
| Orphaned (D-08) | Read-only row: jump disabled + reason badge; text + cached excerpt stay visible. Never navigates, never errors. |
| Pane placement | **LOCKED to the center pane.** The review panel claims the center pane (where `DiffPanel` lives); jump is a single-pane swap (panel → diff → back via the "Review" toggle). This resolves RESEARCH Open Question 1 in favor of the research recommendation; it avoids touching the rightmost `CommitDetail`/`StagingPanel` pane. If the planner finds a blocking layout constraint, escalate — do not silently relocate to the right pane. |
