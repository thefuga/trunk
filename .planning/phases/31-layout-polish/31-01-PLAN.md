# Plan 31-01: Unified Title Bar + Toolbar

**Goal:** Merge the window title bar and toolbar into a single bar by removing native decorations and adding a drag region.

## Wave 1

### Task 31-01-01: Remove native window decorations in Tauri config

<read_first>
- src-tauri/tauri.conf.json (current window configuration)
</read_first>

<action>
In src-tauri/tauri.conf.json, add `"decorations": false` and `"titleBarStyle": "overlay"` to the window config.

Find the windows array item:
```json
"windows": [
  {
    "title": "trunk",
    "width": 800,
    "height": 600
  }
]
```

Change to:
```json
"windows": [
  {
    "title": "trunk",
    "width": 800,
    "height": 600,
    "decorations": false
  }
]
```

This removes the native OS title bar, giving us full control over the window chrome.
</action>

<acceptance_criteria>
- src-tauri/tauri.conf.json window config contains `"decorations": false`
</acceptance_criteria>

### Task 31-01-02: Add drag region and traffic light padding to the unified bar

<read_first>
- src/App.svelte (lines 365-371 — current layout with TabBar and Toolbar in a 36px bar)
- src/components/TabBar.svelte (full file — 29 lines)
</read_first>

<action>
In src/App.svelte, modify the existing bar (line 369) to be the window drag region.

**Step 1: Add data-tauri-drag-region to the bar**

Change line 369:
```svelte
<div class="flex items-center flex-shrink-0" style="height: 36px; background: var(--color-surface); border-bottom: 1px solid var(--color-border);">
```
to:
```svelte
<div data-tauri-drag-region class="flex items-center flex-shrink-0" style="height: 36px; background: var(--color-surface); border-bottom: 1px solid var(--color-border); padding-left: env(titlebar-area-x, 78px);">
```

The `data-tauri-drag-region` attribute makes the bar draggable for window movement.

The `padding-left: env(titlebar-area-x, 78px)` adds space on the left for macOS traffic lights (close/minimize/maximize). On macOS, Tauri positions the traffic lights at ~14px from the left, and they occupy ~54px. A 78px padding ensures content doesn't overlap. On Windows/Linux, this fallback keeps a small margin.

**Step 2: Add drag region to WelcomeScreen**

In src/components/WelcomeScreen.svelte, add a minimal drag region bar at the top so the window is draggable on the welcome screen too.

Read the file first, then add at the very top of the template (before the main content):
```svelte
<div data-tauri-drag-region class="flex-shrink-0" style="height: 36px; background: var(--color-surface); border-bottom: 1px solid var(--color-border); padding-left: env(titlebar-area-x, 78px);"></div>
```
</action>

<acceptance_criteria>
