<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo path (canonicalized) → filename | A canonical repo path could contain `..`, separators, or OS-specific prefixes; it must never be used literally as a filename. |
| on-disk JSON (untrusted) → Rust deserialize | A session file may be corrupt, hand-edited, or written by a newer build; load must never crash or silently destroy data. |
| crash mid-write → durability | A power loss or kill during a write must not leave a truncated/half-written session file. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-65-02-PT | Tampering | session_filename derivation | mitigate | Hash the canonical path with FNV-1a into a `{:016x}.json` token; the filename can contain no `..`/separators/`\\?\` prefix, so a hostile or odd repo path cannot write outside `app_data_dir/sessions/` (Pitfall 1, ASVS V12). |
| T-65-02-DOS | Denial of Service | load_session on corrupt JSON | mitigate | `Result`-based load, never `unwrap`; unparseable file → `serde_json::Value` peek fails → quarantine to `.corrupt` and return `RecoveredCorrupt` (D-15, ASVS V5). The app starts a fresh session instead of crashing. |
| T-65-02-DL | Tampering / data loss | newer schema_version downgrade | mitigate | Peek `schema_version` before full deserialize; if `> 1`, return `RefusedNewer` and leave the file byte-untouched (D-16). A downgraded build can never silently overwrite data written by a newer build. Verified by `newer_version_refused` asserting unchanged bytes. |
| T-65-02-AV | (Availability) | atomic write durability | mitigate | tmp-in-same-dir + `sync_all` + `fs::rename` (D-10, Pitfall 5). A crash mid-write leaves the old file intact or the new file complete — never a truncated session. `atomic_write_clean` asserts no `.tmp` residue. |
| T-65-02-SC | Tampering | npm/cargo installs | mitigate | No new packages (RESEARCH Package Legitimacy Audit: N/A — `serde`, `serde_json`, `std`, `tempfile` already present). No install task; nothing to gate. |
</threat_model>
