<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| frontend invoke (repo path string) → command | The path the frontend passes must be a currently-open repo; a closed/foreign path must be rejected, not operated on. |
| canonical path → ReviewSessionsState key | Two path strings for the same repo (symlink/alias) must map to one key so they cannot create divergent sessions. |
| in-memory state vs. disk | A mutation must hit disk before in-memory state updates, so a failed write cannot leave memory and disk diverged. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-65-03-AC | Elevation / improper access | lifecycle commands on a closed repo | mitigate | Every `_inner` checks `state_map` membership and returns `not_open` TrunkError if the repo is not in `RepoState` (SESS-01 "currently open repository"), mirroring stash.rs:11-19. A command cannot operate on a repo the user has not opened. |
| T-65-03-DIV | Tampering / data integrity | in-memory vs disk divergence | mitigate | Disk-first mutation ordering (D-10 / RESEARCH Pattern 3): `_inner` writes the file → only then does the command insert into `ReviewSessionsState` → then emits. A failed write returns Err before any in-memory change. Status reads merge disk (`file_exists`) with in-memory (`contains_key`) so a stale half cannot misreport `active`. |
| T-65-03-DL | Tampering / data loss | close_repo deleting the session file | mitigate | `close_repo`/`force_close_repo` remove ONLY the in-memory entry; they never call `delete_session`. Asserted by grep + the `resume_after_restart` test. Only `end_review_session` (SESS-03) hard-deletes (D-13). |
| T-65-03-DG | Tampering / data loss | resume of a newer-schema file | mitigate | The resume command branches on `LoadOutcome::RefusedNewer` → returns a `newer_version` error and does NOT create a fresh session, so a downgrade cannot wipe newer data (D-16; the file-untouched guarantee is enforced in Plan 02's `load_session`). |
| T-65-03-SC | Tampering | npm/cargo installs | mitigate | No new packages (RESEARCH Package Legitimacy Audit: N/A). No install task; nothing to gate. |
</threat_model>
