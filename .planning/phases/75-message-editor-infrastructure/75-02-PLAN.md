<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Phase 76 caller → `editor::prepare(message: &str)` | Caller passes a default commit message string built by the backend (`.git/MERGE_MSG` contents, or a constructed merge/revert message). String is content destined for a commit, not arbitrary untrusted input. |
| Helper → filesystem | Helper writes two files under `std::env::temp_dir()`. Files are owned by the host process user; default Unix perms apply. |
| Helper script → git subprocess | When Phase 76 spawns git with `cmd.env("GIT_EDITOR", handle.script_path())`, git execs the script which `cp`s the msg file into git's `$1` (the COMMIT_EDITMSG argument). The script content is `#!/bin/sh\ncp "<msg>" "$1"\n` with `<msg>` quoted. |
| Helper → Drop cleanup | On drop, both files are removed via `std::fs::remove_file`. Errors are swallowed (`let _ = ...`) — Drop cannot return Result. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-75-T01 | Tampering (TOCTOU) | temp file paths | mitigate | Use `tempfile::Builder` to obtain non-predictable paths under `std::env::temp_dir()`. Never construct paths like `/tmp/trunk-editor.sh` manually. `tempfile`'s internal entropy + O_EXCL semantics defend the create-then-write race.
