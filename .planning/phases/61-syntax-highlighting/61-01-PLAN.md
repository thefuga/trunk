**3d. `highlight_line_tokens` function** (per SYNT-01, SYNT-02):
```rust
pub fn highlight_line_tokens(content: &str, extension: &str) -> Vec<SyntaxToken>
```
- Look up syntax via `SYNTAX_SET.find_syntax_by_extension(extension)`, fallback to `find_syntax_plain_text()`
- Get theme from `THEME_SET.themes["base16-ocean.dark"]` (or whichever bundled theme name works)
- Create `HighlightLines::new(syntax, theme)`
- Normalize content: append `\n` if missing (syntect requires newline-terminated lines with `load_defaults_newlines`)
- Call `highlighter.highlight_line(&normalized, &SYNTAX_SET).unwrap_or_default()`
- Iterate the returned `Vec<(Style, &str)>`, tracking byte offset:
  - For each `(style, text)` segment, compute `start = offset`, `end = offset + text.len()`
  - Clamp `end` to `content.len()` (original, without added newline) to avoid overrun
  - Call `color_to_css_class(style.foreground)` to get the CSS class
  - If CSS class is non-empty AND the token's effective length > 0, push a `SyntaxToken { start, end, scope: class.to_string() }`
  - Skip tokens with empty class (default text color) -- they are fillers
  - Advance offset by `text.len()` (using original text len, not clamped)
- Return the token vec

Import `SyntaxToken` from `super::types::SyntaxToken`.

**3e. `merge_spans` function** (per D-08):
```rust
use super::types::{MergedSpan, SyntaxToken, WordSpan};

pub fn merge_spans(
    syntax_tokens: &[SyntaxToken],
    word_spans: &[WordSpan],
    content_len: u32,
) -> Vec<MergedSpan>
```
Algorithm (sweep-line over all boundaries):
1. Collect all boundary points: 0, content_len, and every start/end from both arrays
2. Sort and dedup boundaries
3. For each consecutive pair `(start, end)` where `start < end` and `start < content_len`:
   - Clamp `end` to `min(end, content_len)`
   - Find syntax class: search `syntax_tokens` for a token where `t.start <= start && t.end >= end`, take its `scope`, default to `""`
   - Find emphasis: search `word_spans` for a span where `w.start <= start && w.end >= end && w.emphasized`, return bool
   - Push `MergedSpan { start, end, syntax_class, emphasized }`
4. Return the vec

CRITICAL: The merged span array MUST cover bytes 0 to content_len with no gaps. Every sub-range gets a span even if syntax_class is empty and emphasized is false.

// Pass 2: word-level diff enrichment

for fd in &mut file_diffs {
    let ext = syntax::extension_from_path(&fd.path);
    for hunk in &mut fd.hunks {
        let word_spans_per_line = compute_word_spans_for_hunk(&hunk.lines);
