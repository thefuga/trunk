<objective>
Fix audit finding 66/WR-03: `seed_review_range` currently clones state_map, jumps into `spawn_blocking`, then does `canonical_repo_path + Repository::open + Oid::from_str ×2 + validate_range + compute_range_oids` — and only AFTER the full walk does `seed_review_range_rmw` discover the session is missing and return `no_session`. Empty sessions HashMap → expensive walk → useless error.

Purpose: add a cheap session-existence probe before the walk. Resolve `canonical_repo_path` OUTSIDE `spawn_blocking` (matching siblings like `add_review_commit:676-677`), then briefly lock the sessions mutex to `contains_key` the canonical. If absent, return `no_session` immediately. This also closes INT-W1 (the stylistic inconsistency the audit flagged) by aligning canonical-resolution order with siblings — surface that closure in the SUMMARY.

Output: Rust unit test asserting `no_session` returns fast (no git walk); existing `seed_review_range` happy-path tests still pass.
</objective>
