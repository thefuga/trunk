<objective>
Fix two coupled tech-debt findings in `CommitGraph.svelte`'s review-session machinery by replicating the canonical pattern Phase 73-01 introduced in `ReviewPanel.svelte`.

- **66/WR-01:** the session-changed listener guards on `sessionStatus && event.payload !== sessionStatus.canonical_path`, which is null-permissive — when `sessionStatus` is `null` (initial mount, or after cold transition) the event fires `reloadSession()` regardless of which repo emitted it. Cross-repo events trigger spurious reloads.
- **66/WR-02:** `reloadSession`'s bare `catch { sessionStatus = null; sessionOids = new Set(); }` swallows every IPC failure as if it were `no_session`. A real backend / IPC failure becomes a silent empty graph instead of a toast.

Purpose: pattern-replicate the Phase 73-01 fix (`canonicalPath` tracked separately from `sessionStatus`, fail-closed when null) and branch the catch on `isTrunkError(e) && (e.code === "no_session" || e.code === "not_open")`. Two bugs share one function; bundling avoids two-touch on `reloadSession` and threads the `canonicalPath = null` reset that 73-01 already established as the canonical cleanup.

Output: `CommitGraph.svelte` carries the same listener+reload shape as `ReviewPanel.svelte`; two new vitest cases pin the behavior.
</objective>
