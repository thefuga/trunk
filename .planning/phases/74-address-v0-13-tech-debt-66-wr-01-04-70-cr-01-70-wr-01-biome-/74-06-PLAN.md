<objective>
Fix audit finding 70/CR-01 — the only correctness fix in Phase 74, kept isolated in Wave 3.

`src-tauri/src/git/review.rs:175-204` slices a multi-hunk diff into an excerpt around `[anchor.start_line, anchor.end_line]`. The current line callback inside `diff.foreach` keeps every opposing-side line (the `-` rows when `side == New`, `+` rows when `side == Old`) regardless of which hunk it belongs to. In a multi-hunk file, opposing-side deletions/additions from non-anchored hunks bleed into the excerpt.

Purpose: gate opposing-side lines on per-hunk positional overlap with `[start_line, end_line]`. The clean shape is `git2::Patch::from_diff(&diff, 0)` + iterate `patch.num_hunks()` + per-hunk `hunk_overlaps` decision (RESEARCH §5). This API is already in use in `commands/staging.rs:370,430,481`, confirming git2 0.19 supports it — Assumption A1 from RESEARCH is verified at plan time and the RefCell fallback is NOT needed.

Output: a new failing Rust test demonstrating the leak, then a refactored `slice_diff` that passes it without regressing the three existing single-hunk tests.
</objective>

<interfaces>
Current shape (src-tauri/src/git/review.rs:147-214):
- `slice_diff(repo, anchor) -> Result<String, ExcerptError>`
- Opens commit, builds `DiffOptions` with `opts.pathspec(&anchor.file_path)`, calls `diff_tree_to_tree`.
- Walks via `diff.foreach(file_cb, None, hunk_cb, Some(line_cb))`.
- The line_cb keeps a line if either: (a) its lineno is in `[start_line, end_line]`, OR (b) it has no lineno AND `(side, line.origin())` matches the opposing-direction marker. Branch (b) has no positional gate — this is the bug.

`git2::Patch::from_diff(&diff, 0) -> Result<Option<Patch>, Error>` (verified by usage at staging.rs:370). The `0` is the file index — fine here because `opts.pathspec` constrains the diff to a single file, so index 0 is the only file in the diff. If `Patch::from_diff` returns `None` (file unchanged), surface as `ExcerptError::NoHunks` (matching the existing post-loop `if out.is_empty()` behavior).

Hunk API (git2 0.19): `patch.num_hunks() -> usize`, `patch.hunk(idx) -> Result<(DiffHunk, usize), Error>` returning `(hunk, line_count_in_hunk)`. Hunk metadata: `hunk.new_start()`, `hunk.new_lines()`, `hunk.old_start()`, `hunk.old_lines()` (all `u32`). Line API: `patch.line_in_hunk(hunk_idx, line_idx) -> Result<DiffLine, Error>`, with `.new_lineno() -> Option<u32>`, `.old_lineno() -> Option<u32>`, `.origin() -> char`, `.content() -> &[u8]`.

Existing helpers (RESEARCH §D): `make_repo`, `commit_with_file`, etc. in `src-tauri/src/git/review.rs:586-650` — reuse them. Do not extract a shared helper module (IN-02 out of scope).
