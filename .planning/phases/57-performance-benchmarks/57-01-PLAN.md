**Create `src-tauri/benches/bench_commands.rs`:**

Per D-02: Required targets: `list_refs_inner`, `diff_unstaged_inner`, `stage_hunk_inner`.
Per D-03 discretion: Also benchmark `get_status_inner` (HIGH priority -- called every poll cycle, exercises `repo.statuses()`).
Per D-09: Use `OnceLock` for read-only benchmarks. Per pitfall 2: Use `iter_batched` for `stage_hunk_inner` (mutating operation).

The file must contain:

**Fixture helpers:**
- A `BenchRepo` struct (same as bench_graph.rs -- holding `_dir: tempfile::TempDir`, `path: std::path::PathBuf`)
- `make_repo_with_branches(branch_count: usize) -> BenchRepo`: Creates a repo with initial commit on main, then creates `branch_count` branches each with 2 additional commits. Produces a repo with refs for `list_refs_inner` to enumerate. Use `repo.branch("branch-N", &commit, false)` after committing.
- `make_repo_with_unstaged_changes() -> BenchRepo`: Creates a repo with initial commit containing `README.md` with "initial content", then writes "modified content" to `README.md` on the filesystem (using `std::fs::write`). This produces an unstaged diff for `diff_unstaged_inner` and `get_status_inner`.
- `make_repo_for_stage_hunk() -> (tempfile::TempDir, String, HashMap<String, PathBuf>)`: Creates a repo with initial commit, then modifies README.md to produce an unstaged hunk. Returns (dir, path_string, state_map). This is for `iter_batched` -- a fresh one per iteration since staging is mutating.

**state_map construction pattern:**
For each fixture, build `HashMap<String, PathBuf>` where key = `dir.path().display().to_string()` and value = `dir.path().to_path_buf()`. This is the pattern that `open_repo_from_state` expects.

**OnceLock fixtures (read-only benchmarks):**
- `static REPO_BRANCHES: OnceLock<BenchRepo>` -- 50 branches (for `list_refs_inner`)
- `static REPO_UNSTAGED: OnceLock<BenchRepo>` -- repo with unstaged changes (for `diff_unstaged_inner` and `get_status_inner`)

**Benchmark functions:**

1. `bench_list_refs(c: &mut Criterion)`:
   - Get or init `REPO_BRANCHES` with `make_repo_with_branches(50)`
   - Build state_map from the repo path
   - `c.bench_function("list_refs_inner", |b| { b.iter(|| trunk_lib::commands::branches::list_refs_inner(&path, &state_map).unwrap()) })`

2. `bench_diff_unstaged(c: &mut Criterion)`:
   - Get or init `REPO_UNSTAGED` with `make_repo_with_unstaged_changes()`
   - Build state_map
   - `c.bench_function("diff_unstaged_inner", |b| { b.iter(|| trunk_lib::commands::diff::diff_unstaged_inner(&path, "README.md", &state_map).unwrap()) })`

3. `bench_get_status(c: &mut Criterion)`:
   - Reuse `REPO_UNSTAGED` (same fixture -- get_status reads, doesn't mutate)
   - Build state_map
   - `c.bench_function("get_status_inner", |b| { b.iter(|| trunk_lib::commands::staging::get_status_inner(&path, &state_map).unwrap()) })`

4. `bench_stage_hunk(c: &mut Criterion)`:
   - Use `iter_batched` with `BatchSize::SmallInput` per pitfall 2 (mutating operation)
   - Setup closure: call `make_repo_for_stage_hunk()` returning `(dir, path, state_map)`
   - Routine closure: call `trunk_lib::commands::staging::stage_hunk_inner(&path, "README.md", 0, &state_map).unwrap()` then `drop(dir)`

- `criterion_group!(benches, bench_list_refs, bench_diff_unstaged, bench_get_status, bench_stage_hunk);`
- `criterion_main!(benches);`
