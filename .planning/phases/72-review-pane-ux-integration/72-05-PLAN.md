<objective>
Three-task gap closure to close the Phase 72 UAT failures discovered after the verifier ran:

1. **Drop the Cmd+Shift+R accelerator.** User reports a clash with their launcher tool (Recents) and prefers no shortcut for code review at all. The View menu entry stays — only the keyboard binding goes. REQ-72-1b is retracted, not satisfied; VERIFICATION.md and VALIDATION.md must reflect that.

2. **Restore the ReviewPanel scroll.** 72-04's RepoView cleanup collapsed two layout wrappers and changed `height: 100%` → `flex: 1`. The parent `.flex-1.overflow-hidden` is a flex *child* (no `display:flex`), so `flex: 1` resolves to intrinsic height and the comments list has no constrained height to scroll against. Restoring `height: 100%; min-height: 0` brings back pre-72 sizing behavior in a single attribute change.

3. **Record the lifecycle gap as Phase 73 carry-forward.** Manual UAT also surfaced Bug 3 (comments not visible on first ReviewPanel open after app boot) and the user's design ask (no "End review" affordance — the review feels permanent). Auto-resuming the session without a corresponding End-review action would compound the permanence problem rather than fix it. Both belong in Phase 73 (review-lifecycle-management); this plan only records the carry-forward.

Purpose: Each task is independent and surgical (one Rust line, one CSS attribute, three docs). Bundled into one plan because they're all "close out Phase 72 against actual UAT" rather than scope creep. No new tests — Bug 1 is OS config, Bug 2 is visual layout (covered by manual UAT). The existing test suite must still pass (`just check` is the gate).
