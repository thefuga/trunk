## Typography

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Tab label | 12px | 500 (medium) | 1.0 (line-height: normal within 32px bar) | Tab name text, truncated at max-width 200px |
| Tab label (active) | 12px | 500 (medium) | 1.0 | Same size, differentiated by color not weight |

Rationale: This phase only introduces tab bar text. All text lives in the 32px title bar. A single size (12px) at medium weight matches the existing TabBar and Toolbar patterns. No headings, no body text, no display text needed.

## Color

All colors reference existing CSS custom properties from `app.css`. No new color tokens needed.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Title bar background | `--color-surface` | #161b22 | Tab bar container background (existing) |
| Tab text (inactive) | `--color-text-muted` | #8b949e | Inactive tab label color |
| Tab text (active) | `--color-text` | #c9d1d9 | Active tab label color |
| Active tab background | `--color-bg` | #0d1117 | Active tab gets the main background color to visually "connect" to the content area below |
| Tab separator | `--color-border` | #30363d | 1px right border between tabs |
| Tab close hover | `--color-border` | #30363d | Close button hover background (existing pattern) |
| Dirty indicator | `--color-accent` | #388bfd | 6px diameter dot badge on tabs with uncommitted changes |
| New tab button (+) | `--color-text-muted` | #8b949e | Plus icon color, same as inactive tab text |
| New tab button hover | `--color-border` | #30363d | Hover background for + button |

Color distribution within the tab bar:
- 60% dominant: `--color-surface` (the tab bar strip background)
- 30% secondary: `--color-bg` (active tab background, connecting to content)
- 10% accent: `--color-accent` (dirty dot badge only)

Accent reserved for: dirty indicator dot badge only. Never for active tab underline, tab text, or close button.

## Component Inventory

New and modified components for this phase:

### TabBar.svelte (FULL REWRITE)

**Layout:** Horizontal flex container inside the 32px title bar. Sits between the traffic-light padding (left) and Toolbar (right).

**Structure:**
```
[traffic light pad 78px] [tab1] [tab2] [tab3] ... [+ button] [--- drag spacer ---] [Toolbar]
```

**Tab item anatomy:**
```
[repo-name (truncate 200px)] [dirty-dot? (6px)] [close-btn (16x16)]
```

**Dimensions:**
- Tab bar height: 32px (inherits from title bar)
- Tab horizontal padding: 10px left + 10px right
- Tab internal gap: 4px between elements
- Close button: 16x16px, 4px border-radius, centered X icon at 12px
- Dirty dot: 6px width, 6px height, border-radius 50%
- New tab (+) button: 24x24px, centered Plus icon at 14px
- Tab max-width: none (natural content width, truncate label at 200px)
- Tab min-width: none (tabs keep natural width per D-06)

**States:**
| State | Background | Text Color | Border |
|-------|-----------|------------|--------|
| Inactive | transparent | `--color-text-muted` | 1px right `--color-border` |
| Active | `--color-bg` | `--color-text` | 1px right `--color-border` |
| Hover (inactive) | `rgba(255,255,255,0.04)` | `--color-text-muted` | 1px right `--color-border` |

**Overflow (D-06):** When tabs exceed available width, the container scrolls horizontally. Hidden scrollbar (`scrollbar-width: none`, `::-webkit-scrollbar { display: none }`). Scroll via mouse wheel, trackpad, or programmatic scroll-into-view on tab activation.

**Drag region:** The spacer div between the last tab/+ button and Toolbar has `data-tauri-drag-region`. Tab buttons and + button do NOT have drag region. Empty areas of the tab bar container DO have drag region.

**Interactions:**
| Action | Trigger | Behavior |
|--------|---------|----------|
| Switch tab | Click tab | Set `activeTabId` to clicked tab's ID |
| Close tab | Click X button | Remove tab, call `close_repo` if repo open. If last tab, show single empty tab. |
| Force close tab | Shift+click X button | Call `force_close_repo` (D-11) then remove tab |
| New tab | Click + button | Append new tab with `repoPath: null`, switch to it, show WelcomeScreen |

### RepoView.svelte (NEW)

Not a visual component — it is the extracted content area from App.svelte. Contains all per-repo state and renders the existing layout (sidebar + graph + staging/diff panels). No new visual elements. Uses `display: contents` when active, `display: none` when inactive (keep-alive per D-08).

### WelcomeScreen.svelte (UNCHANGED)

Reused as-is for empty tabs (tabs with `repoPath: null`). No visual changes needed.

## Interaction Contract

### Keyboard Shortcuts

| Shortcut | Action | Scope |
|----------|--------|-------|
| Cmd+T | Create new tab (empty, shows WelcomeScreen) | App-level |
| Cmd+W | Close active tab (graceful, D-10) | App-level |
| Cmd+Shift+W | Force close active tab (D-11) | App-level |
| Cmd+1 through Cmd+9 | Switch to tab by position (1-indexed, Cmd+9 = last tab) | App-level |
| Ctrl+Tab | Switch to next tab (wraps around) | App-level |
| Ctrl+Shift+Tab | Switch to previous tab (wraps around) | App-level |

**Cmd+9 special case:** Always switches to the last tab regardless of count (Chrome/VS Code convention).

**Last tab close:** When the last tab is closed, a new empty tab is created automatically so the window is never empty.

### Tab Persistence (TAB-06)

On every tab change (open, close, switch, repo loaded in tab):
- Persist `open_tabs: PersistedTab[]` and `active_tab_id: string` to LazyStore
- Debounce writes by 500ms to avoid excessive I/O during rapid switching

On app launch:
- Restore tabs from store. If empty, show single empty tab.
- Validate each persisted repo path via `open_repo`. If path invalid, remove tab.
- Migrate legacy `open_repo` single-value to tab array (one-time migration).

### Dirty Indicator (TAB-07)

- A 6px diameter circle in `--color-accent` (#388bfd) appears between the tab label and the close button
- Appears when `staged + unstaged > 0` for that tab's repo (D-04)
- Driven by `repo-changed` watcher events + `get_dirty_counts` backend call (D-05)
- Disappears when all changes are committed or discarded
- No animation on appear/disappear — instant show/hide

## Copywriting Contract

| Element | Copy |
|---------|------|
| Empty tab label | "New Tab" |
| + button aria-label | "New tab" |
| Close button aria-label | "Close tab" |
| Tab with repo label | Repository directory name (e.g., "trunk", "my-project") — basename of repo path |
| Last tab close behavior | No confirmation — a fresh empty tab replaces it silently |

**Empty state:** When a tab has no repo open, the WelcomeScreen is shown (existing component, unchanged copy). The tab label reads "New Tab".

**Error state:** If a persisted tab's repo path no longer exists on disk at app launch, the tab is silently removed. No error toast needed — the repo simply is not there anymore. If all restored tabs fail, user sees a single empty tab with WelcomeScreen.

**Destructive actions in this phase:**
| Action | Confirmation Approach |
|--------|----------------------|
| Close tab (X button) | No confirmation — graceful close, running ops finish naturally (D-10) |
| Force close tab (Shift+X) | No confirmation dialog — the Shift modifier is the explicit intent signal (D-11) |
| Close last tab | No confirmation — auto-creates empty tab |

Rationale: Tabs are lightweight, non-destructive to close. The repo data is on disk. Unsaved commit message loss is acceptable (same as VS Code/browser tab behavior).

## Phase-Specific Visual Details

### Title Bar Layout (32px)

```
|<-- 78px traffic lights -->|<-- tab bar (scrollable) -->|<-- spacer (drag) -->|<-- toolbar -->|
```

- Total height: 32px
- Background: `--color-surface` (#161b22)
- Bottom border: 1px solid `--color-border` (#30363d)
- Traffic light padding: `padding-left: {78 / zoomLevel}px` (existing)

### Active Tab Visual Connection

The active tab uses `--color-bg` (#0d1117) background — the same as the main content area below. This creates a visual "tab sticks out from the page" effect without needing a bottom-border trick. The inactive tabs sit on `--color-surface` and the active tab's background matches the content, making it appear connected.

No bottom border removal needed. The title bar's bottom border continues across inactive tabs. The active tab's darker background naturally reads as "this tab is selected."

### Tab Close Button

- Size: 16x16px (w-4 h-4)
- Border-radius: 2px (rounded-sm)
- Icon: Lucide `X` at 12px
- Color: `--color-text-muted`
- Hover: background `var(--color-border)`, slightly visible on the surface
- Transition: `transition-colors` (Tailwind utility)
- `onclick` uses `stopPropagation` to prevent triggering tab switch

### New Tab Button (+)

- Size: 24x24px
- Border-radius: 4px
- Icon: Lucide `Plus` at 14px
- Color: `--color-text-muted`
- Hover: background `var(--color-border)`
- Position: After the last tab, before the drag spacer
- `flex-shrink: 0` to prevent collapsing
