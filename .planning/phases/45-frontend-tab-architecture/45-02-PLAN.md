    **Tab CRUD functions:**
    ```typescript
    function addNewTab() {
      const tab: TabInfo = {
        id: createTabId(),
        repoPath: null,
        repoName: 'New Tab',
        dirty: false,
      };
      tabs = [...tabs, tab];
      activeTabId = tab.id;
      persistTabs();
    }

    function openRepoInTab(tabId: string, path: string, name: string) {
      const tab = tabs.find(t => t.id === tabId);
      if (tab) {
        tab.repoPath = path;
        tab.repoName = name;
        persistTabs();
      }
    }

    function closeTab(tabId: string) {
      const idx = tabs.findIndex(t => t.id === tabId);
      if (idx === -1) return;
      const tab = tabs[idx];

      // Remove tab from array
      tabs = tabs.filter(t => t.id !== tabId);

      // If last tab closed, add empty tab
      if (tabs.length === 0) {
        addNewTab();
        return;
      }

      // If active tab was closed, switch to adjacent
      if (activeTabId === tabId) {
        const newIdx = Math.min(idx, tabs.length - 1);
        activeTabId = tabs[newIdx].id;
      }

      // Close repo on backend (after removing from DOM to prevent stale effects)
      if (tab.repoPath) {
        safeInvoke('close_repo', { path: tab.repoPath }).catch(() => {});
      }

      persistTabs();
    }

    function forceCloseTab(tabId: string) {
      const tab = tabs.find(t => t.id === tabId);
      if (!tab) return;
      const repoPath = tab.repoPath;

      // Remove tab from array first
      const idx = tabs.findIndex(t => t.id === tabId);
      tabs = tabs.filter(t => t.id !== tabId);

      if (tabs.length === 0) {
        addNewTab();
      } else if (activeTabId === tabId) {
        const newIdx = Math.min(idx, tabs.length - 1);
        activeTabId = tabs[newIdx].id;
      }

      // Force close on backend (D-11)
      if (repoPath) {
        safeInvoke('force_close_repo', { path: repoPath }).catch(() => {});
      }

      persistTabs();
    }
    ```

    **IMPORTANT architectural note about Toolbar placement:**
    The Toolbar is currently in the title bar. With RepoView extraction, the Toolbar must still render in the title bar row. Two options:
    1. Toolbar stays in App.svelte title bar and receives the active tab's remoteState/undoRedo via a lookup
    2. Toolbar moves into RepoView and gets positioned in the title bar via CSS

    Choose option 2: Move Toolbar into RepoView.svelte but keep it inside the title bar. Actually, this is architecturally tricky because the title bar is in App.svelte but Toolbar needs per-tab state from RepoView.

    The cleanest approach: The title bar in App.svelte contains only the TabBar and a drag spacer. The Toolbar is part of RepoView.svelte, rendered at the top of RepoView's content. RepoView's first element is a title-bar-height flex container for the Toolbar, then the main content below. But this means each tab has its own Toolbar instance (which is correct for per-tab state). The issue is that the title bar background/border-bottom is in App.svelte and the Toolbar needs to visually be part of it.

    Better approach: Keep the title bar structure in App.svelte. Pass the active tab's RepoView a way to render Toolbar into the title bar slot. Or simpler: make RepoView expose its remoteState and undoRedo so App.svelte can pass them to Toolbar.

    SIMPLEST approach (and most aligned with keep-alive): Each RepoView contains its own Toolbar. The App.svelte title bar contains ONLY TabBar + drag spacer. The Toolbar moves inside `<main>` of each RepoView, rendered at the top. This means:
    - The 32px title bar = TabBar only
    - Below the title bar, each RepoView starts with its own Toolbar row
    - Toolbar is no longer in the title bar but below it

    BUT this changes the visual layout. Currently Toolbar is IN the title bar. The UI spec says: `[traffic light pad 78px] [tab1] [tab2] [tab3] ... [+ button] [--- drag spacer ---] [Toolbar]`

    So the Toolbar must remain in the title bar. To do this cleanly:
    - RepoView creates remoteState and undoRedo internally
    - RepoView exposes them via bind:this or a callback
    - App.svelte gets the active tab's remoteState/undoRedo and passes to Toolbar

    Actually the simplest: store a Map of per-tab state in App.svelte. When creating each tab, create the remoteState/undoRedo in App.svelte and pass them down to RepoView as props. This way App.svelte has access to them for the Toolbar too.
