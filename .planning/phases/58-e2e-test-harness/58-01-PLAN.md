**e2e/wdio.conf.js:**
Adapted from the official Tauri v2 WebdriverIO example (RESEARCH.md "Complete wdio.conf.js for Trunk" section). Key details:
- `host: '127.0.0.1'`, `port: 4444`
- `specs: ['./specs/**/*.e2e.js']`
- `maxInstances: 1` (both top-level and per-capability)
- `capabilities[0]['tauri:options'].application` set to `path.resolve(__dirname, '../src-tauri/target/debug/trunk')`
- `reporters: ['spec']`
- `framework: 'mocha'` with `mochaOpts: { ui: 'bdd', timeout: 60000 }`
- `onPrepare` hook: runs `spawnSync('bun', ['run', 'tauri', 'build', '--', '--debug', '--no-bundle'], { cwd: path.resolve(__dirname, '..'), stdio: 'inherit', shell: true })` -- NOTE: only do this if the binary doesn't already exist or if `E2E_SKIP_BUILD` env var is not set. Add a check: `if (!process.env.E2E_SKIP_BUILD) { ... build ... }`. This allows CI to build separately and skip the onPrepare build.
- `beforeSession` hook: spawns `tauri-driver` at `path.resolve(os.homedir(), '.cargo', 'bin', 'tauri-driver')` with `stdio: [null, process.stdout, process.stderr]`. Sets up `on('error')` and `on('exit')` handlers per RESEARCH.md example.
- `afterSession` hook: calls `closeTauriDriver()` which sets `exit = true` then kills the process.
- `onShutdown` handler for SIGINT/SIGTERM/SIGHUP/SIGBREAK cleanup per RESEARCH.md example.

**e2e/helpers/fixture.js:**
Export functions for creating test fixture git repos using `child_process.execSync` and `fs` APIs:

- `createLinearRepo(commitCount = 3)`: Creates a temp dir via `mkdtempSync(join(tmpdir(), 'trunk-e2e-'))`, runs `git init`, configures `user.email "test@test.com"` and `user.name "Test"`, then loops `commitCount` times creating `file-{i}.txt` with `content {i}`, `git add .`, `git commit -m "commit {i}"`. Returns the directory path.

- `createBranchRepo()`: Creates a temp dir, inits git, configures user, creates initial commit with README.md, then creates branches `feature-a` and `feature-b` via `git branch`. Returns directory path.

- `createDirtyRepo()`: Creates a linear repo with 1 commit, then writes `dirty-file.txt` without staging. Returns directory path. (Needed for staging tests.)

- `cleanupRepo(dir)`: Removes the temp directory via `rmSync(dir, { recursive: true, force: true })`.

All functions import from `child_process`, `fs`, `path`, and `os`.

**e2e/helpers/app.js:**
Export functions for interacting with the Trunk app from WebdriverIO context:

- `openRepo(repoPath)`: Calls `await browser.execute(async (path) => { await window.__TAURI_INTERNALS__.invoke('open_repo', { path }); }, repoPath)`. Note: If `browser.execute` doesn't handle async properly, fall back to `browser.executeAsync((path, done) => { window.__TAURI_INTERNALS__.invoke('open_repo', { path }).then(() => done()).catch(done); }, repoPath)`.

- `waitForCommitGraph(timeout = 10000)`: Calls `const row = await $('[data-testid="commit-row"]'); await row.waitForExist({ timeout });`.

- `waitForBranchSidebar(timeout = 10000)`: Calls `const sidebar = await $('[data-testid="branch-sidebar"]'); await sidebar.waitForExist({ timeout });`.

After creating all files, run `cd e2e && bun install` to install dependencies and generate the lockfile.
