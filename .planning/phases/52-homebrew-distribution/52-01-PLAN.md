  <action>
Extend `.github/workflows/release.yml` by appending two new jobs after the existing `build` job. Do NOT modify the existing `build` job at all.

**Job 1: `publish`**
- `needs: [build]`
- `runs-on: ubuntu-latest`
- `permissions: contents: write` (inherits from top-level, but be explicit)
- Single step that uses `gh release edit "${{ github.ref_name }}" --draft=false --repo "${{ github.repository }}"` with `GITHUB_TOKEN`
- Per D-02: this auto-publishes the draft release created by tauri-action

**Job 2: `update-tap`**
- `needs: [publish]`
- `runs-on: ubuntu-latest`
- `if: ${{ !contains(github.ref_name, '-') }}` -- skip for prerelease tags (per research recommendation)
- Steps:
  1. `actions/checkout@v6` -- needed to read `src-tauri/tauri.conf.json`
  2. **Get Tauri version** -- `id: tauri`, run: `echo "version=$(jq -r .version src-tauri/tauri.conf.json)" >> "$GITHUB_OUTPUT"`
  3. **Download DMGs and compute checksums** -- env `GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}`, run:
     ```
     TAG="${{ github.ref_name }}"
     gh release download "$TAG" --pattern "*.dmg" --dir ./dmgs
     echo "arm_sha256=$(shasum -a 256 ./dmgs/*_aarch64.dmg | awk '{print $1}')" >> "$GITHUB_ENV"
     echo "intel_sha256=$(shasum -a 256 ./dmgs/*_x64.dmg | awk '{print $1}')" >> "$GITHUB_ENV"
     ```
  4. **Generate cask and push to tap** -- env `HOMEBREW_TAP_TOKEN: ${{ secrets.HOMEBREW_TAP_TOKEN }}`, run:
     ```
     TAG="${{ github.ref_name }}"
     VERSION="${TAG#v}"
     TAURI_VERSION="${{ steps.tauri.outputs.version }}"

     git clone "https://x-access-token:${HOMEBREW_TAP_TOKEN}@github.com/joaofnds/homebrew-tap.git" tap

     cat > tap/Casks/trunk.rb << 'CASK'
     # Generated by joaofnds/trunk release workflow. DO NOT EDIT.
     cask "trunk" do
       version "PLACEHOLDER_VERSION"

       on_intel do
         sha256 "PLACEHOLDER_INTEL_SHA256"
         url "https://github.com/joaofnds/trunk/releases/download/vPLACEHOLDER_VERSION/trunk_PLACEHOLDER_TAURI_VERSION_x64.dmg"
       end
       on_arm do
         sha256 "PLACEHOLDER_ARM_SHA256"
         url "https://github.com/joaofnds/trunk/releases/download/vPLACEHOLDER_VERSION/trunk_PLACEHOLDER_TAURI_VERSION_aarch64.dmg"
       end

       name "Trunk"
       desc "Desktop Git GUI"
       homepage "https://github.com/joaofnds/trunk"

       app "trunk.app"

       zap trash: [
         "~/Library/Application Support/com.joaofnds.trunk",
         "~/Library/Caches/com.joaofnds.trunk",
         "~/Library/WebKit/com.joaofnds.trunk",
       ]
     end
     CASK
     ```
     Then use `sed` to replace placeholders:
     ```
     sed -i "s/PLACEHOLDER_VERSION/${VERSION}/g" tap/Casks/trunk.rb
     sed -i "s/PLACEHOLDER_TAURI_VERSION/${TAURI_VERSION}/g" tap/Casks/trunk.rb
     sed -i "s/PLACEHOLDER_INTEL_SHA256/${intel_sha256}/g" tap/Casks/trunk.rb
     sed -i "s/PLACEHOLDER_ARM_SHA256/${arm_sha256}/g" tap/Casks/trunk.rb
     ```
     Then commit and push:
     ```
     cd tap
     git config user.name "github-actions[bot]"
     git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
     git add Casks/trunk.rb
     git commit -m "Update trunk to ${TAG}"
     git push
     ```

**Key constraints per locked decisions:**
- Per D-01: Formula update automated via release workflow
- Per D-02: Auto-publish draft release after builds
- Per D-05: Use on_intel/on_arm blocks (not arch shorthand)
- Per D-06: Hardcoded DMG naming pattern `trunk_{TAURI_VERSION}_{ARCH}.dmg`

**Critical version note:** The git tag (e.g., v0.10.0) and Tauri version (e.g., 0.1.0) are different. The cask `version` field uses the git tag without the `v` prefix. The DMG filename uses the Tauri version from tauri.conf.json. Both are needed in the URL.
