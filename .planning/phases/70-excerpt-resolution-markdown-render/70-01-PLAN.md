<objective>
Build the pure Rust markdown renderer at `src-tauri/src/git/review.rs`. Reads a `&ReviewSession` + `&git2::Repository`, re-resolves each anchored comment fresh from git2 (full-file blob slice OR diff replay slice), formats the doc per D-03/D-04/D-05/D-06/D-07/D-08, fence-tags excerpts per D-10/L-07, and routes every resolution failure into the unresolvable section per L-04/L-09. Returns a single `String`; never panics.

Purpose: All the content rules from DOC-02/03/04 are content-shape decisions on a deterministic pure function. The renderer is the canonical TDD candidate — defined I/O, no I/O side effects, golden-test friendly. Doing this in TDD locks behavior before format details have a chance to drift.

Output: One new module (`src-tauri/src/git/review.rs`) with `pub fn render`, an internal render-only `ExcerptError` enum, leaf helpers (`fence_length`, `fence_language`, `slice_full_file`, `slice_diff`), and an embedded `#[cfg(test)] mod tests` suite covering the ten golden behaviors derived from RESEARCH §Validation Architecture.
</objective>
