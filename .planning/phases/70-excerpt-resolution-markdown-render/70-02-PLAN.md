Wire the pure renderer from 70-01 into Tauri's IPC layer. Adds one read-only command `generate_review_doc(path: String)` that loads the in-memory `ReviewSession` for the canonical repo path, enforces D-11 (≥1 comment), opens a fresh `git2::Repository` inside `spawn_blocking`, calls `crate::git::review::render(&session, &repo)`, and returns the markdown `String`. Purpose: The command is the only place D-11 can be enforced — the pure renderer is L-04 + assume-≥1. Keeping the gate one layer up means the renderer stays panic-free without a defensive zero-comment branch. The command is the IPC contract for DOC-01: "User can generate one markdown document from the session."

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tauri IPC: TS `safeInvoke("generate_review_doc", {path})` -> Rust command | `path` is user-controlled but constrained: must be a key already in `RepoState`'s open-repo map. `canonical_repo_path` enforces this. |
| `git2::Repository::open(&path)` inside spawn_blocking | The opened repo is the user's; libgit2 handles its own input validation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-70-02-PATH | Tampering | `path` arg to generate_review_doc | mitigate | `canonical_repo_path` rejects any path not in `RepoState`. An attacker would need to first open a malicious path through the open-repo flow (out of scope here). Same control as every other review command. |
| T-70-02-LOCK | Denial of Service | `ReviewSessionsState` mutex | mitigate | Lock is held only for the duration of the `session.clone()` — never across `spawn_blocking`. Mirrors the pattern from `resolve_session_comments` (Pitfall 6). |
| T-70-02-SYNC | Tampering / DoS | `git2::Repository` Send/Sync | mitigate | `Repository::open` is called INSIDE the `spawn_blocking` closure — never stashed in state. `cargo clippy` catches `Send`/`Sync` violations; `just check` gates this. |
| T-70-02-EMIT | Denial of Service | event emission storms | mitigate | Command explicitly does NOT emit `session-changed`. Read-only paths never emit. |
| T-70-02-D11 | Tampering | zero-comment renderer invariant | mitigate | D-11 gate at the command layer enforces `session.comments.len() >= 1` before `render()` is called; the renderer assumes the invariant and has no defensive branch. Any non-UI invocation that bypasses the UI gate hits the command-layer gate. |

Package legitimacy: N/A — no new dependencies.
