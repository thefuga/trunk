## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Chevron-to-label gap, action button padding |
| sm | 8px | Row horizontal padding (matches existing FileRow `padding: 0 8px`) |
| md | 16px | Tree indentation per nesting level (`depth * 16px`) |
| lg | 24px | Section header height (matches existing 24px panel header) |
| xl | 32px | Not used in this phase |

Exceptions:
- **26px row height**: Both FileRow and DirectoryRow use 26px height (established project convention, not on the 4px grid but locked by existing FileRow)
- **6px gap**: FileRow uses `gap: 6px` between icon and text; DirectoryRow matches this for visual consistency
- **12px panel header padding**: StagingPanel header uses `padding: 0 12px`; toggle button lives inside this space

## Interaction Contract

### Expand/Collapse (TREE-04)

| Trigger | Action |
|---------|--------|
| Click directory row | Toggle expand/collapse |
| Click chevron icon | Toggle expand/collapse |
| ArrowRight on collapsed directory | Expand |
| ArrowLeft on expanded directory | Collapse |
| Toggle flat-to-tree | All directories start collapsed (D-09) |
| Status refresh | Expanded set preserved; tree data re-derives (D-08, TREE-05) |

### Keyboard Navigation (TREE-06, D-11)

| Key | Context | Action |
|-----|---------|--------|
| ArrowDown | Any row | Move focus to next visible row |
| ArrowUp | Any row | Move focus to previous visible row |
| ArrowRight | Collapsed directory | Expand directory |
| ArrowRight | Expanded directory | Move focus to first child |
| ArrowRight | File | No-op |
| ArrowLeft | Expanded directory | Collapse directory |
| ArrowLeft | Collapsed directory | Jump focus to parent directory |
| ArrowLeft | File | Jump focus to parent directory |
| Enter | File | Select file (trigger diff display / onfileselect callback) |
| Enter | Directory | No-op |

Focus management:
- Focus is per-section: unstaged, staged, conflicted, commit-files each track independently (D-07, D-13)
- Focus is an index into the flattened visible row array
- On data change (status refresh): if previously focused path still exists in new flat array, update index to match; otherwise clamp to valid range (D-13)
- Focus resets to 0 when switching from flat to tree mode
- Active in both flat and tree modes (D-12)

### View Mode Toggle (TREE-01, TREE-02, TREE-03)

| Property | Value |
|----------|-------|
| Scope | Global — one toggle controls all file list contexts (D-01, D-03) |
| Persistence | `tree_view_enabled` boolean in LazyStore (D-02) |
| Default | `false` (flat mode) |
| Consumers | StagingPanel (unstaged, staged, conflicted), CommitDetail file list, MergeEditor conflict file selector |
| Data flow | RepoView loads from store on mount, passes as prop; StagingPanel header toggle updates store + notifies parent via callback |

## ARIA & Accessibility Contract

| Element | ARIA |
|---------|------|
| Tree container | `role="tree"` when in tree mode, `role="list"` when in flat mode |
| DirectoryRow | `role="treeitem"`, `aria-expanded={expanded}`, `aria-level={depth + 1}` |
| FileRow (tree mode) | `role="treeitem"`, `aria-level={depth + 1}` |
| FileRow (flat mode) | `role="listitem"` (existing) |
| Keyboard focus | `tabindex="0"` on the container, arrow keys navigate within |
| Toggle button | `role="switch"`, `aria-checked={treeViewEnabled}`, `aria-label` per copywriting contract |

## Animation Contract

| Element | Animation |
|---------|-----------|
| Expand/collapse | **Instant** (no animation). Matches existing StagingPanel section expand/collapse behavior. |
| Toggle switch | **Instant**. Icon swaps immediately between List and FolderTree. |
| Focus highlight | **Instant**. Background color applied immediately on focus change. |

Rationale: The flatten-then-render approach rebuilds the flat array on every expand/collapse. Animating individual rows entering/leaving would require tracking enter/exit transitions on a dynamic list, adding significant complexity for minimal UX gain. Existing section expand/collapse in StagingPanel is instant, so this is consistent.
