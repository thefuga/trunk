The root cause: each diff line div has `overflow-x: {wordWrap ? 'hidden' : 'auto'}` which creates per-line horizontal scrollbars. The fix is to remove per-line overflow and let the parent container handle horizontal scrolling.

**DiffViewer.svelte (line 76):**
Change the wrapper div from `overflow-y: auto` to `overflow: auto` so it scrolls both horizontally and vertically when content overflows. This is the single scroll container for all inline views (HunkView, FullFileView).

**HunkView.svelte (line ~314):**
Remove the `overflow-x: {wordWrap ? 'hidden' : 'auto'}` from each diff line div's inline style. The line's `white-space: pre` (when wordWrap is off) will cause lines to extend beyond the container width, and the parent DiffViewer container will provide the single horizontal scrollbar.

**FullFileView.svelte (line ~76):**
Same fix as HunkView -- remove `overflow-x: {wordWrap ? 'hidden' : 'auto'}` from each diff line div's inline style.

**SplitView.svelte (line 336):**
The `.split-cell` has `overflow-x: auto`. For split view, the two panels should scroll horizontally independently (per Phase 64 decision: "independent horizontal scroll per panel"). The fix here is different: keep independent horizontal scroll but move it from cell-level to a wrapper around all rows for each side. However, given the current DOM structure where each row has a left cell and right cell side by side, the simplest correct approach is: keep `overflow-x: auto` on `.split-cell` since split view cells are already independent panels. The SplitView already works correctly with per-cell horizontal scroll because each cell IS a panel in the split layout. No change needed for SplitView.

Actually, re-examining SplitView: each `.split-row` contains two `.split-cell` divs. Each cell contains one line. So per-cell overflow-x is effectively per-line overflow-x, the same problem. But for split view the Phase 64 decision says "independent horizontal scroll per panel". The solution: remove `overflow-x: auto` from `.split-cell`, and instead let the DiffViewer parent handle it for split view too. Since left and right are in the same row, horizontal scrolling of the parent will move both sides together. If the user wants independent horizontal scroll (per Phase 64), we'd need a different DOM structure. For now, since the user's bug report is about scrolling the "entire view" together, apply the same fix: remove `overflow-x: auto` from `.split-cell` and let DiffViewer handle it.

Summary of changes:
1. `DiffViewer.svelte`: Change `overflow-y: auto` to `overflow: auto`
2. `HunkView.svelte`: Remove `overflow-x: {wordWrap ? 'hidden' : 'auto'};` from diff line inline style
3. `FullFileView.svelte`: Remove `overflow-x: {wordWrap ? 'hidden' : 'auto'};` from diff line inline style
4. `SplitView.svelte`: Change `.split-cell` from `overflow-x: auto` to `overflow-x: hidden` (parent DiffViewer handles scroll)
