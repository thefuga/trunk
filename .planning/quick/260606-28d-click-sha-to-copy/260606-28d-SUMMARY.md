# Quick Task 260606-28d — Summary

## Outcome

Every passive SHA display in the UI is now click-to-copy: clicking copies the **full**
40-char oid and confirms with a toast. The existing right-click "Copy SHA" context menu
now routes through the same helper, so left- and right-click behave identically (both toast).

## What changed

**New helper** — `src/lib/clipboard.ts`
- `copySha(oid)`: `writeText(oid)` → `showToast("Copied <short>")`; on failure
  `showToast("Failed to copy: <msg>", "error")`. Never rejects. Always copies the full oid.

**Wired 5 display sites** (each renders the SHA in a reset-styled `<button>` with
`title="Copy SHA"` + hover underline, calling `copySha(fullOid)`):
1. `CommitRow.svelte` — SHA column cell; `stopPropagation` on click + keydown so it no
   longer selects the row.
2. `CommitDetail.svelte` — toolbar `commit: <sha>` (full `commitDetail.oid`).
3. `CommitDetail.svelte` — `parent: <sha>` (full `commitDetail.parent_oids[0]`).
4. `RebaseEditor.svelte` — SHA column cell; `stopPropagation` on click + keydown + dblclick
   so it no longer focuses/edits the row.
5. `ReviewPanel.svelte` — the commit-group SHA button now **copies**; its previous
   jump-to-commit action moved onto the commit **summary** (no navigation lost).

**Consistency tweak** — `CommitGraph.svelte` right-click "Copy SHA" now calls `copySha`
(previously a silent `writeText(...).catch()`), so it gives the same toast.

## Conscious exclusions (not SHA displays)

- `RebaseEditor` `baseName` toolbar pill (from RepoView `rebaseBaseName`): a ref label that
  is a branch name when resolvable and only a short SHA as a fallback; the full oid is not
  threaded to that component. Not a canonical SHA display.
- `DiffPanel.commitDetail?.oid`: used only as comment-anchor data, never rendered as text.

## Tests

- `src/lib/clipboard.test.ts` (new): copies full oid; success toast; error toast on failure;
  never rejects (real toast store, mock only the third-party `writeText`).
- Extended `CommitRow.test.ts`, `CommitDetail.test.ts`, `RebaseEditor.test.ts` with
  copy-the-full-oid + no-select/no-focus assertions (fixtures discriminate full vs short oid).
- `ReviewPanel.test.ts`: renamed the existing "short oid jumps" test → "summary jumps", added
  "short oid copies the full oid"; anchored the doc-Copy-button query (`/^Cop(y|ied)$/`) so it
  no longer collides with the new "Copy SHA" label.
