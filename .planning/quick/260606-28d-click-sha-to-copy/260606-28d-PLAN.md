# Quick Task 260606-28d: Click a displayed SHA to copy it

## Goal

Everywhere the UI shows a commit SHA (or short SHA), clicking it copies the **full**
40-char oid to the clipboard and confirms with a toast.

## Locked decisions (from discussion)

- **Scope:** every passive SHA display becomes click-to-copy.
- **Row conflict:** in clickable rows, the SHA copies *only* — it must not also
  select/focus the row (stop propagation on click **and** keydown).
- **Feedback:** toast confirmation via the existing `showToast` helper.
- **Length:** copy the full oid, even though a short form is displayed.
- **ReviewPanel:** its SHA is already a "jump to commit" button — move that jump
  onto the commit summary so the SHA itself can copy (no navigation lost).

## Display sites (5)

| # | File | Site | Full oid source | In clickable parent? |
|---|------|------|-----------------|----------------------|
| 1 | CommitRow.svelte | SHA column cell | `commit.oid` | yes (row select) |
| 2 | CommitDetail.svelte | toolbar `commit: <sha>` | `commitDetail.oid` | no |
| 3 | CommitDetail.svelte | `parent: <sha>` | `commitDetail.parent_oids[0]` | no |
| 4 | RebaseEditor.svelte | SHA column cell | `item.oid` | yes (row focus/dblclick) |
| 5 | ReviewPanel.svelte | commit-group SHA button | `group.oid` | no (jump moves to summary) |

## Conscious exclusions

- **RebaseEditor `baseName` pill** (RepoView `rebaseBaseName`): a ref label that is a
  branch name when resolvable and only a short SHA as a *fallback*; the full oid is not
  threaded to that component. Not a canonical SHA display — excluded.
- **DiffPanel `commitDetail?.oid`**: used only as comment-anchor data, never rendered as
  visible text — excluded.

## Approach

Extract one helper `copySha(oid)` in `src/lib/clipboard.ts`:
- `writeText(oid)` then `showToast("Copied <short>")`; on failure `showToast("Failed to
  copy: <msg>", "error")` (mirrors the existing ReviewPanel copy-error wording).

Each site renders the SHA inside a reset-styled `<button>` (whole SHA cell for rows,
`w-full text-left`), `title="Copy SHA"`, hover underline for affordance, calling
`copySha(fullOid)`. In rows: `e.stopPropagation()` on click, keydown, and dblclick.

Also route CommitGraph's existing right-click "Copy SHA" through `copySha` so left- and
right-click give the same (now toasted) behavior.

## Tasks

1. **Helper + unit test** — `clipboard.ts` + `clipboard.test.ts` (real toast store,
   mock `writeText`): copies full oid + success toast; error toast on failure.
2. **Wire 5 sites + extend component tests** — each asserts `writeText(fullOid)`;
   row sites assert no select/focus; ReviewPanel: SHA copies, summary jumps
   (update the existing "short oid jumps" test → summary jumps).
3. **Route right-click Copy SHA through `copySha`** (CommitGraph).
