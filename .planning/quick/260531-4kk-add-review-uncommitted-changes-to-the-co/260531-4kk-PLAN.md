<objective>
Let the WORKING TREE be a review target alongside hand-picked commits. Capture the
current uncommitted changes into a REAL but DANGLING commit (parent = HEAD) and add
that commit's OID to the active review session exactly like a hand-picked commit. The
entire existing pipeline (intersect_graph_order, classify_anchor, render, persistence)
then works UNCHANGED — `Side::Old` resolves against the parent tree (= HEAD), `Side::New`
against the snapshot tree (= working tree), which is precisely "before vs after" for
uncommitted work.

Purpose: review uncommitted work in the same manual review-session → markdown flow
without adding any LLM call, new Source/Side variant, or schema-version bump.
Output: a `snapshot_working_tree` git2 helper, one new `add_working_tree_review`
Tauri command, an additive optional session field to track the single snapshot, and a
"Review uncommitted changes" control in the review panel.

Design is LOCKED (do not re-litigate): snapshot-at-capture (immutable), one combined
HEAD→workdir target (staged + unstaged + untracked-but-NOT-ignored), GC degradation
accepted (no keep-alive ref), re-snapshot REPLACES the prior snapshot.
</objective>
