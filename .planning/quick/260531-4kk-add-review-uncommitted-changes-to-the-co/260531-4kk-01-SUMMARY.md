# Quick 260531-4kk Plan 01: Review uncommitted changes Summary

The working tree is now a review target: a "Review uncommitted changes" control captures the current uncommitted changes (staged + unstaged + untracked-not-ignored) into a real dangling commit (parent = HEAD) and adds it to the active session, flowing through the unchanged render/anchor/persist pipeline with `Side::Old` = HEAD and `Side::New` = working tree.

## What was built

- **Task 1 — `snapshot_working_tree` helper + schema field** (`c54cd2d`): New `src-tauri/src/git/workdir_snapshot.rs` with `pub fn snapshot_working_tree(repo) -> Result<Oid, TrunkError>`. Builds a throwaway `Index::new()` + `set_index`, `add_all(["*"], DEFAULT)` (respects .gitignore), `write_tree_to` (ODB only, never `index.write()`), parent = HEAD or none when unborn, dangling commit (`None` ref). 4 unit tests: untracked-included, ignored-excluded, real `.git/index` byte-for-byte untouched, unborn-HEAD parent-less. Added `#[serde(default)] pub working_tree_snapshot: Option<String>` to `ReviewSession`.
- **Task 2 — `add_working_tree_review` command** (`c6bed14`): `set_working_tree_snapshot_rmw` does remove-old + add-new + field-update in one `mutate_session_rmw` closure (at most one snapshot, never stacked). Command runs the snapshot in `spawn_blocking` (git2 not Sync), has a `no_session` fast-fail precheck, then the atomic RMW + `emit_session_changed`. Registered in `lib.rs`. Unit test proves replace-not-stack and pointer tracking.
- **Task 3 — panel control** (`fe5b679`): `reviewWorkingTree(repoPath)` safeInvoke wrapper on `ReviewSessionManager`; header button gated on `sessionState !== "none"`, error surface mirrors `onEndClick` (errorMessage/showToast). Snapshot commit appears via the existing session-changed → reload path. Theme CSS tokens only.
