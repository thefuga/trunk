# Quick 260531-l02: Hunk-level Comment button in the no-selection toolbar — Summary

A "Comment" button now appears in the NO-SELECTION hunk toolbar (unstaged, beside
Stage Hunk; commit, merge-disabled) and comments the whole hunk's New-side range by
synthesizing a full-hunk selection and delegating to the existing line-level path —
reusing the composer, working-tree snapshot, and Old-side scope guard with zero
duplicated logic.

## What was built (Task 1)

1. **`hunkSelectableIndices(hunk: DiffHunk): Set<number>`** (`src/lib/diff-anchor.ts`) —
   pure helper returning the indices of every non-context line, the same
   `origin !== 'Context'` convention the diff views use for click targets. Unit-tested
   for mixed Add/Delete/Context, pure-add, pure-delete, and pure-context hunks
   (`src/lib/diff-anchor.test.ts`, 4 new cases).

2. **`handleCommentHunk(filePath, hunkIndex)`** (`src/components/DiffPanel.svelte`) —
   finds the FileDiff + hunk, builds the full-hunk selection via
   `hunkSelectableIndices`, sets `selectedHunkKey` + `selectedLineIndices`, then
   `await handleCommentLines(...)`. This reuses the `isMerge` guard, the unstaged
   Old-side scope guard (pure-delete → toast + return), `ensureActiveSession`, the
   `ensure_working_tree_snapshot` fetch, and the composer verbatim. A degenerate hunk
   with no selectable lines (empty set) opens nothing.

3. **`oncommenthunk` prop** threaded `DiffPanel → DiffViewer → {HunkView, SplitView}`
   (required prop, mirroring `oncommentlines`).

4. **Buttons** (HunkView + SplitView):
   - Unstaged no-selection `{:else}` branch: accent "Comment" button beside Stage Hunk.
   - Commit branch: new `{:else}` (no-selection) arm with the accent "Comment" button
     carrying the SAME `disabled={isMerge}` + merge tooltip as the line-level commit
     Comment button.
   - Staged branch and FullFileView untouched. No inline colors — reused the existing
     accent markup (HunkView) / `staging-btn accent-btn` class (SplitView).

## UAT note (expected behavior, not a bug)

On a pure-deletion hunk in unstaged mode (UAT step 5), `handleCommentHunk` synthesizes
the full-hunk selection *before* delegating, so when `handleCommentLines` trips the
Old-side guard and returns via toast, the synthesized selection stays visible (lines
highlighted, toolbar shows "Comment (N)") until Escape/next click. This is consistent
with the existing line-level behavior (line-level also leaves the selection after its
toast) and satisfies the "no-op + toast" must-have. Deliberately NOT fixed by
pre-checking `resolveSide` in `handleCommentHunk` — that would duplicate the guard and
break the zero-duplication delegation design.
