# Summary: Remove the bottom bar and use the new notification system for state updates

## What changed

The 26px bottom status bar (`StatusBar.svelte`) was removed. All remote operation feedback (progress, success, errors) now flows through the existing toast notification system.

## Task 1: Move remote-progress listener and enhance toast feedback

- Added `listen('remote-progress', ...)` effect to `Toolbar.svelte` so `remoteState.progressLine` is still updated during remote operations
- Added `errorMessage()` helper to both `Toolbar.svelte` and `PullDropdown.svelte` for detailed error messages in toasts (auth failure, non-fast-forward, etc.)
- Updated `Toolbar.runRemote` to show detailed error messages via `errorMessage()` instead of generic strings
- Added `showToast` calls to `PullDropdown.svelte`'s `runRemote` for success and error on Fetch, Pull (ff/ff-only/rebase) operations

## Task 2: Remove StatusBar from App.svelte

- Removed `import StatusBar` and `<StatusBar repoPath={repoPath!} />` from App.svelte
- Main content now fills the full window height without the 26px bottom bar

## Task 3: Delete StatusBar.svelte

- Deleted `src/components/StatusBar.svelte`
- Verified zero dangling references in `src/` (only a comment in Toolbar mentioning the relocation origin)

## Verification

- `npm run check` passes with no new errors
- `remoteState` module kept intact — Toolbar/PullDropdown still use `isRunning` to disable buttons during remote ops
