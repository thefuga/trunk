<objective>
Extend computeGraphSvgData to generate SVG path data for sentinel rows (WIP and stash) instead of skipping them, using TDD.

Purpose: The data layer must produce path entries with a `dashed` flag for sentinel rows so that GraphCell (Plan 02) can render them with dashed stroke-dasharray and differentiated dot shapes. Currently, `computeGraphSvgData` explicitly skips all `__`-prefixed OIDs. This plan replaces that skip with sentinel-specific path generation.

Output: Updated `SvgPathData` type, `computeGraphSvgData` function, and comprehensive test coverage for sentinel path generation.
</objective>

  <files>src/lib/types.ts, src/lib/graph-svg-data.ts, src/lib/graph-svg-data.test.ts</files>
  <behavior>
    WIP connector path:
    - Input: GraphCommit with oid='__wip__' at rowIndex=0, column=0, color_index=0
    - Expected: Map entry keyed `__wip__:connector:0` with:
      - d: `M ${cx(0)} ${cy(0) + DOT_RADIUS} V ${rowBottom(0)}` (from just below WIP dot to row bottom)
      - colorIndex: 0
      - dashed: true
    - No other path entries for __wip__ (no straight edges, no rails)

    Stash connector path:
    - Input: GraphCommit with oid='__stash_0__' at rowIndex=2, column=0, color_index=0, edges with Straight from_column=0 to_column=0
    - Expected: Map entry keyed `__stash_0__:connector:0` with:
      - d: `M ${cx(0)} ${cy(2) + DOT_RADIUS} V ${rowBottom(2)}` (from just below stash dot to row bottom)
      - colorIndex: 0
      - dashed: true
    - Stash should also generate its incoming straight edges (pass-through rails from other lanes)

    dashed flag:
    - Sentinel paths: dashed = true
    - All existing non-sentinel paths: dashed = undefined (no change)

    Existing behavior preserved:
    - Non-sentinel commits produce same paths as before
    - Key format unchanged for non-sentinel paths
    - colorIndex propagation unchanged

    Edge case — stash with pass-through edges:
    - Input: __stash_0__ with Straight edge from_column=1, to_column=1 (another lane passing through)
    - Expected: Both the dashed connector path AND the normal straight edge for column 1
    - The pass-through straight edge should NOT be dashed (it belongs to another lane)
  </behavior>
  <implementation>
    1. Add `dashed?: boolean` to SvgPathData interface in types.ts
    2. In computeGraphSvgData, replace the sentinel skip block with:
       - If oid === '__wip__': Generate one dashed connector path from dot center + DOT_RADIUS down to rowBottom. Key: `{oid}:connector:{column}`. Set dashed: true. Then continue (WIP has no real edges).
       - If oid.startsWith('__stash_'): Generate one dashed connector path (same pattern as WIP). Then fall through to normal edge processing for any pass-through straight edges (but do NOT mark those as dashed).
    3. Import DOT_RADIUS in graph-svg-data.ts (currently not imported)
  </implementation>
