**2. GraphCell.svelte -- new component consuming graphSvgData via context:**

Create `src/components/GraphCell.svelte` with:

- Props: `commit: GraphCommit`, `rowIndex: number`, `maxColumns: number` (default 1)
- Use `getContext` to retrieve the `graphSvgData` context object
- Compute `svgWidth = Math.max(maxColumns, commit.column + 1) * LANE_WIDTH`
- Filter paths from the Map: iterate all entries, select those whose key starts with `commit.oid + ':'`. This captures all straight, connection, and rail paths for this commit's row.
- Categorize paths: keys containing `:straight:` or `:rail:` are rail paths (stroke-linecap="butt"); all others are connection paths (stroke-linecap="round").
- Compute dot position: `cx = commit.column * LANE_WIDTH + LANE_WIDTH / 2`, `dotCy = rowIndex * ROW_HEIGHT + ROW_HEIGHT / 2` (MUST use global Y coordinates to match the viewBox).
- Render SVG with viewBox clipping:
  ```svelte
  <svg width={svgWidth} height={ROW_HEIGHT} viewBox="0 {rowIndex * ROW_HEIGHT} {svgWidth} {ROW_HEIGHT}" style="overflow: hidden; flex-shrink: 0;">
    <!-- Layer 1: Rails (straight/rail paths) -->
    {#each railPaths as path}
      <path d={path.d} fill="none" stroke={laneColor(path.colorIndex)} stroke-width={EDGE_STROKE} stroke-linecap="butt" />
    {/each}

    <!-- Layer 2: Connection edges -->
    {#each connectionPaths as path}
      <path d={path.d} fill="none" stroke={laneColor(path.colorIndex)} stroke-width={EDGE_STROKE} stroke-linecap="round" />
    {/each}

    <!-- Layer 3: Commit dot -->
    {#if commit.is_merge}
      <circle cx={dotCx} cy={dotCy} r={DOT_RADIUS} fill="var(--color-bg)" stroke={laneColor(commit.color_index)} stroke-width={MERGE_STROKE} />
    {:else}
      <circle cx={dotCx} cy={dotCy} r={DOT_RADIUS} fill={laneColor(commit.color_index)} />
    {/if}
  </svg>
  ```

- Use `const laneColor = (idx: number) => \`var(--lane-${idx % 8})\`;` (same as LaneSvg)

**3. CommitRow.svelte -- conditional rendering:**

- Import `GraphCell` from `'./GraphCell.svelte'`
- Add `rowIndex: number` to the Props interface (required, no default)
- In the graph column (lines 101-105), replace the unconditional `<LaneSvg>` with:
  ```svelte
  {#if commit.oid.startsWith('__')}
    <LaneSvg {commit} {maxColumns} />
  {:else}
    <GraphCell {commit} {rowIndex} {maxColumns} />
  {/if}
  ```

**Key correctness details:**
- The viewBox uses `rowIndex * ROW_HEIGHT` as the Y offset -- this MUST match the absolute Y coordinates used by `computeGraphSvgData`. The rowIndex comes from `displayItems` (which includes WIP row when present), matching exactly what `computeGraphSvgData` receives.
- Do NOT use `overflow: visible` on GraphCell's SVG. The old LaneSvg uses `overflow: visible` because paths bleed into adjacent rows. GraphCell uses `overflow: hidden` because viewBox handles coordinate mapping.
- Path filtering by OID prefix is correct because `computeGraphSvgData` keys every path with its commit's OID. Each row only needs its own commit's paths.
- Do NOT add any path computation logic in GraphCell -- all path data comes from the pre-computed Map.
