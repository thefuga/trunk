# Requirements: Trunk

**Defined:** 2026-03-15
**Core Value:** A developer can open any Git repository, browse its full commit history as a visual graph, stage files, and create commits — all without touching the terminal.

## v0.6 Requirements

Requirements for milestone v0.6 UI Polish & Core Ops. Each maps to roadmap phases.

### Icons

- [x] **ICON-01**: App uses Lucide SVG icons replacing Unicode symbols across all components (Toolbar, FileRow, StagingPanel, CommitForm, BranchSidebar, TabBar)

### Git Operations

- [x] **GITOP-01**: User can discard changes for an individual unstaged file with confirmation dialog
- [x] **GITOP-02**: User can discard all unstaged changes at once with confirmation dialog
- [x] **GITOP-03**: User can delete a local branch via right-click context menu with confirmation
- [x] **GITOP-04**: User can delete a tag via right-click context menu with confirmation
- [x] **GITOP-05**: User can rename a local branch via right-click context menu
- [x] **GITOP-06**: User can reset current branch to any commit (soft/mixed/hard) via context menu

### Staging & Commit UX

- [x] **STAGE-01**: Commit form has a three-way selector (commit / amend / stash) replacing the amend checkbox
- [x] **STAGE-02**: In stash mode, commit form subject auto-populates as the stash name
- [x] **STAGE-03**: "Stage all changes" button is styled green
- [x] **STAGE-04**: "Unstage all changes" button is styled red
- [x] **STAGE-05**: Unstaged and staged file lists render at equal height when both expanded

### Graph

- [x] **GRAPH-01**: Commit graph has visible padding above first and below last commit row
- [x] **GRAPH-02**: Graph column can shrink below graph content width, compressing lanes
- [x] **GRAPH-03**: Clicking a branch or tag in sidebar scrolls graph to that commit's row

### Bug Fixes

- [x] **FIX-01**: New untracked files included in dirty counts and trigger WIP row in graph
- [x] **FIX-02**: Last visible column header does not render a trailing resize divider

### Toast System

- [x] **TOAST-01**: App displays toast notifications for operation success and error feedback

### Layout

- [x] **LAYOUT-01**: Right pane auto-opens when user clicks a commit, branch, or navigable item while pane is closed
- [x] **LAYOUT-02**: Window top bar and tab/actions bar are visually merged into a single bar

## Future Requirements

Deferred to future milestones. Tracked but not in current roadmap.

### v0.7 — Hunk Staging & Search

- **HUNK-01**: User can stage/unstage individual hunks within a file diff
- **SRCH-01**: User can search commit graph for hashes, messages, and branches with cmd+f

### v0.8 — Conflict & Rebase

- **CNFL-01**: User can view conflict diffs for files with merge conflicts
- **CNFL-02**: User can resolve conflicts within the app
- **RBASE-01**: User can perform interactive rebase

### v0.9 — Multi-tab & Views

- **TAB-01**: User can open multiple repositories in separate tabs
- **VIEW-01**: User can toggle between list view and preview for file lists

### v1.0 — Infrastructure

- **TEST-01**: End-to-end test harness drives the application from real entry points (GOOS-style)
- **PERF-01**: Benchmark suite covers core algorithms and critical paths

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Custom titlebar (decorations: false) | CSS-only bar merge is sufficient for v0.6; full custom titlebar has platform-specific risks |
| Remote branch delete | Local-only delete for v0.6; remote delete requires push --delete with auth handling |
| Undo discard (stash-before-discard) | Adds complexity; discard is intentionally destructive with confirmation |
| Better tag pill icon in SVG overlay | Deferred — requires SVG path data injection into overlay, low priority |
| Overflow pill z-index fix | Not selected for this milestone |
| Settings/preferences UI | Deferred to v1.0 |
| Commit signing | Deferred to v1.0 |
| Auto-updates | Deferred to v1.0 |
