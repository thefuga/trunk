    In the `graphOverlay` snippet in `src/components/CommitGraph.svelte` (around line 432), the SVG element has `class="absolute top-0 left-0"`. Replace `left-0` with a dynamic inline style.

    Current:
    ```svelte
    <svg
      class="absolute top-0 left-0"
      width={Math.max(maxColumns, 1) * OVERLAY_LANE_WIDTH}
      height={contentHeight}
      style="pointer-events: none; z-index: 1;"
    >
    ```

    Change to:
    ```svelte
    <svg
      class="absolute top-0"
      width={Math.max(maxColumns, 1) * OVERLAY_LANE_WIDTH}

    ```

    The `columnWidths.ref` value is already reactive state in CommitGraph (`let columnWidths = $state<ColumnWidths>({ ref: 120, ... })`). Using it in the style attribute makes the SVG re-position if the user resizes the ref column.

    Gap root cause: The virtual list scroll container starts at x=0 of the full row, but graph content begins after the ref column (~120px). `left-0` placed the SVG at the start of the row, making dots appear offset to the left beside the graph rather than over it.
  </action>
