<feature>
  <name>buildOverlayPaths — SVG path generation for overlay edges</name>
  <files>src/lib/types.ts, src/lib/overlay-paths.ts, src/lib/overlay-paths.test.ts</files>
  <behavior>
    ## Rail paths (CURV-02)
    - Rail edge (fromX=0, fromY=0, toX=0, toY=3) → `M 8 0 V 144` (rowTop(0) to rowBottom(3))
    - Rail with branch tip at start (fromY node is branch tip) → starts at cy(fromY) not rowTop(fromY)
    - Rail with branch tip at end (toY node is branch tip) → ends at cy(toY) not rowBottom(toY)
    - Rail carries colorIndex and dashed flag through
    - Rail has kind='rail'

    ## Connection paths (CURV-01)
    - Connection edge (fromX=0, toX=2, fromY=3) → SVG path with M, H, C, V commands
    - Path starts at cx(fromX), cy(fromY) — dot center of source
    - Path goes horizontal toward target column
    - At target column: 90° rounded corner using cubic bezier C command
    - Corner radius = OVERLAY_LANE_WIDTH / 2 = 8px (fixed)
    - Kappa constant κ = 4(√2-1)/3 ≈ 0.5522847498 for quarter-circle bezier
    - Direction determination: examine rail edges to infer merge vs fork at target column
      - If a rail in toX starts at the connection row (merge pattern) → corner curves DOWN, vertical to rowBottom
      - If a rail in toX ends at the connection row (fork pattern) → corner curves UP, vertical to rowTop
      - If no rail in toX touches connection row → infer from available rail position
    - Connection carries colorIndex and dashed flag through
    - Connection has kind='connection'

    ## Fixed radius for all distances (CURV-04)
    - Adjacent connection (1 row gap between source/target dots) uses 8px corner radius
    - Distant connection (10+ row gap) uses identical 8px corner radius
    - Only the vertical/horizontal segment lengths change — corner shape is constant

    ## Edge cases
    - Empty edges array → empty result
    - Left-going connections (toX < fromX) produce mirrored paths
    - WIP edges (dashed=true) produce identical geometry with dashed=true flag
    - Multiple connections at same row in different columns produce separate paths
  </behavior>
  <implementation>
    1. Add OverlayPath type to src/lib/types.ts (extends SvgPathData with kind field)
    2. Create src/lib/overlay-paths.ts with:
       - Local coordinate helpers: cx(), cy(), rowTop(), rowBottom() using OVERLAY_* constants
       - Corner radius R = OVERLAY_LANE_WIDTH / 2
       - Kappa constant for bezier approximation
       - buildRailPath() helper — generates M...V paths with branch tip awareness
       - buildConnectionPath() helper — generates M...H...C...V paths with direction-aware corners
       - buildOverlayPaths(data: OverlayGraphData): OverlayPath[] — main entry point
         API change from CONTEXT.md's `buildOverlayPaths(edges: OverlayEdge[])`: needs OverlayGraphData
