    Also import `OverlayRefPill` type if needed.

    **2. Add pill data computation as $derived after paths:**
    ```typescript
    const pillData = $derived.by(() => buildRefPillData(graphData.nodes, displayItems, columnWidths.ref, measureTextWidth));
    ```
    This reacts to `columnWidths.ref` changes (user resizing ref column) per user constraint.

    **3. Add hover state:**
    ```typescript
    let hoveredPill = $state<OverlayRefPill | null>(null);
    ```

    **4. Expand SVG to cover ref + graph columns:**
    Change the SVG element from:
    ```
    width={Math.max(maxColumns, 1) * LANE_WIDTH}
    style="left: {columnWidths.ref}px; pointer-events: none; z-index: 1;"
    ```
    To:
    ```
    width={columnWidths.ref + Math.max(maxColumns, 1) * LANE_WIDTH}
    style="left: 0; pointer-events: none; z-index: 1;"
    ```
    Only apply this expansion when `columnVisibility.ref` is true. When ref column is hidden, keep the SVG positioned at `left: 0` with just the graph width.

    **5. Wrap existing <g> groups with translate for graph offset:**
    Wrap the three existing `<g>` groups (overlay-rails, overlay-connections, overlay-dots) so they offset by the ref column width:
    - Add `transform="translate({columnVisibility.ref ? columnWidths.ref : 0}, 0)"` to each of the three existing `<g>` groups, OR wrap all three in a parent `<g>` with that transform.

    **6. Update getVisibleOverlayElements call to pass pillData:**
    ```typescript
    const visible = getVisibleOverlayElements(paths, graphData.nodes, visibleStart, visibleEnd, pillData);
    ```

    **7. Add fourth <g class="overlay-pills"> layer (after overlay-dots, no translate — pills render in ref column space):**
    Render for each `visible.pills`:

    a. **Connector line** — straight horizontal `<line>` from pill right edge to commit dot:
       - `x1={pill.x + pill.width}`, `y1={pill.y}`
       - `x2={columnWidths.ref + pill.dotCx}` (dotCx is in graph-local coords, need to offset by ref column width since SVG now starts at x=0), `y2={pill.dotCy}`
       - Wait — dotCx is computed as `cx(node.x)` which is graph-local. Since the SVG now starts at x=0, the dot's actual SVG x is `refWidth + dotCx`. So connector goes from `pill.x + pill.width` to `(columnVisibility.ref ? columnWidths.ref : 0) + pill.dotCx`.
       - `stroke={laneColor(pill.commitColorIndex)}`, `stroke-width={EDGE_STROKE}`
       - Apply dimming: `opacity={pill.isRemoteOnly ? 0.67 : 1}`, `style={pill.isNonHead && !pill.isRemoteOnly ? 'filter: brightness(0.75)' : ''}`
       - Only render connector when `columnVisibility.graph` is true (connector needs a dot to connect to)

    b. **Capsule rect** — `<rect>` with fully rounded ends:
       - `x={pill.x}`, `y={pill.y - PILL_HEIGHT / 2}`, `width={pill.width}`, `height={PILL_HEIGHT}`
       - `rx={PILL_HEIGHT / 2}`, `ry={PILL_HEIGHT / 2}` (capsule shape per user decision)
       - `fill={laneColor(pill.colorIndex)}`
       - Dimming: `opacity={pill.isRemoteOnly ? 0.67 : 1}`, `style={pill.isNonHead && !pill.isRemoteOnly ? 'filter: brightness(0.75)' : ''}`
       - `pointer-events="auto"` for hover detection
       - `onmouseenter={() => hoveredPill = pill}`, `onmouseleave={() => hoveredPill = null}`

    c. **SVG path icon** for Tag and Stash ref types (per user decision — replace unicode prefixes):
       - Tag icon: small diamond/tag shape ~10x10 at `pill.x + PILL_PADDING_X`, centered at `pill.y`. Example: `<path d="M {pill.x + PILL_PADDING_X} {pill.y} l 4 -4 l 4 4 l -4 4 z" fill="white" opacity="0.9" />`
       - Stash icon: small flag shape at same position. Example: `<path d="M {pill.x + PILL_PADDING_X} {pill.y + 4} v -8 h 5 v 4 h -5" fill="none" stroke="white" stroke-width="1.2" opacity="0.9" />`
       - Branch refs: no icon, text starts at `pill.x + PILL_PADDING_X`

    d. **Pill text** — `<text>` element:
       - `x={pill.x + PILL_PADDING_X + (pill.refType === 'Tag' || pill.refType === 'Stash' ? ICON_WIDTH : 0)}`
       - `y={pill.y}`
       - `fill="white"`, `font-size={PILL_FONT_SIZE}`, `font-family="var(--font-sans)"`
       - `font-weight={pill.isHead ? 700 : 500}`
       - `dominant-baseline="central"` (critical — prevents vertical misalignment, see Research pitfall 1)
       - Content: `{pill.truncatedLabel}`

    e. **Overflow +N badge** (when `pill.overflowCount > 0`):
       - Compute badge text = `+${pill.overflowCount}`, badge width via measureTextWidth or estimate at `BADGE_FONT_SIZE * 0.7 * badgeText.length + PILL_PADDING_X * 2`
       - `<rect>` at `x={pill.x + pill.width + PILL_GAP}`, `y={pill.y - BADGE_HEIGHT / 2}`, `width={badgeWidth}`, `height={BADGE_HEIGHT}`, `rx={BADGE_HEIGHT / 2}`, `ry={BADGE_HEIGHT / 2}`
       - `fill={laneColor(pill.colorIndex)}`, `style="filter: brightness(0.65)"` (darkened per user decision)
       - `pointer-events="auto"`, same hover handlers as pill rect
       - `<text>` centered in badge: `text-anchor="middle"`, `dominant-baseline="central"`, `font-size={BADGE_FONT_SIZE}`

    **8. Add HTML hover overlay (outside SVG, inside the same scroll container):**
    Place after the `</svg>` closing tag but inside the graphOverlay snippet (it needs to scroll with content). Actually, the graphOverlay snippet returns SVG content. The hover overlay needs to be HTML. Add it as a separate element in the VirtualList's overlay area.

    Better approach: Add the hover overlay as a sibling to the SVG inside the graphOverlay snippet. Since snippets can return multiple elements in Svelte 5, render:
    ```svelte
    {#if hoveredPill}
      <div
        class="absolute rounded-lg shadow-lg"
        style="
          left: {hoveredPill.x}px;
          top: {hoveredPill.y - PILL_HEIGHT / 2}px;
          background: var(--lane-{hoveredPill.colorIndex % 8});
          padding: 4px 8px;
          z-index: 50;
          pointer-events: auto;
          clip-path: inset(0 0 0 0 round 8px);
          opacity: 1;
          transition: clip-path 180ms ease, opacity 120ms ease;
        "
        onmouseenter={() => {/* keep hoveredPill */}}
        onmouseleave={() => hoveredPill = null}
      >
        {#each hoveredPill.allRefs as ref}
          <div class="text-[11px] leading-5 font-medium text-white whitespace-nowrap">
            {ref.short_name}
          </div>
        {/each}
      </div>
    {/if}
    ```

    If the snippet can't return HTML siblings alongside the SVG, place the hover overlay outside the snippet — as a child of the `.flex-1.overflow-hidden` container that wraps VirtualList, positioned absolutely. Use `top` computed from `hoveredPill.y - PILL_HEIGHT / 2` and offset by scroll position. The simplest approach may be to place it after the VirtualList component with absolute positioning.

    Use Claude's discretion per user decision on the exact hover expansion implementation. The key requirements are: smooth 180ms animation, shows all refs one per line, positioned over the pill area.

    **9. Handle edge cases:**
    - When `columnVisibility.ref` is false: skip pill rendering entirely (no pills, no connectors)
    - When `columnVisibility.graph` is false: render pills but no connector lines (dots aren't visible)
    - Ensure hover overlay doesn't break commit row click/context menu interactions
