  <files>src/components/CommitGraph.svelte</files>
  <action>
    Add a proof-of-concept SVG overlay to CommitGraph.svelte:

    **Step 1: Define the overlay snippet.**

    Add a Svelte snippet block in CommitGraph.svelte that creates the SVG overlay:

    ```svelte
    {#snippet graphOverlay(contentHeight: number)}
      <svg
        class="absolute top-0 left-0"
        width={Math.max(maxColumns, 1) * LANE_WIDTH}
        height={contentHeight}
        style="pointer-events: none; z-index: 1;"
      >
        <!-- Proof-of-concept: semi-transparent overlay to verify scroll sync -->
        <rect x="0" y="0" width="100%" height="100%" fill="rgba(255,0,0,0.03)" />
      </svg>
    {/snippet}
    ```

    Use existing `LANE_WIDTH` (12px) for width calculation since the old rendering pipeline is still active. The overlay width matches the current graph column width. Future phases will switch to OVERLAY_LANE_WIDTH.

    The red tint (0.03 opacity) makes the overlay barely visible — enough to confirm it's rendering and scrolling correctly, not enough to interfere visually.

    **Step 2: Pass the snippet to VirtualList.**

    Update the VirtualList usage in the template:

    ```svelte
    <VirtualList
      bind:this={listRef}
      items={displayItems}
      defaultEstimatedItemHeight={ROW_HEIGHT}
      onLoadMore={loadMore}
      loadMoreThreshold={50}
      {hasMore}
      overlaySnippet={graphOverlay}
    >
    ```

    **Step 3: Verify build.**

    Run `npx vite build` and `npx vitest run` to confirm everything compiles and tests pass.
