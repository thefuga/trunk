<objective>
Implement `buildGraphData()` — a pure TypeScript function that transforms Rust's `GraphCommit[]` output into `OverlayGraphData` with integer grid coordinates (x=swimlane column, y=row index) and coalesced vertical edges.

Purpose: This is the data bridge between Rust's lane algorithm and the SVG overlay renderer. Edge coalescing reduces SVG DOM nodes from O(commits × lanes) to O(lanes + merge_edges), critical for performance at 10k+ commits.

Output: `src/lib/active-lanes.ts` (implementation) and `src/lib/active-lanes.test.ts` (comprehensive tests covering all topology scenarios).

  is_merge: boolean;
  is_branch_tip: boolean;
  is_stash: boolean;
}

export interface OverlayNode {
  oid: string;
  x: number;           // swimlane index (column)
  y: number;           // row index
  colorIndex: number;
  isMerge: boolean;
  isBranchTip: boolean;
  isStash: boolean;
  isWip: boolean;
}

export interface OverlayEdge {
  fromX: number;        // source swimlane

  dashed: boolean;
}

Implement `buildGraphData()` in `src/lib/active-lanes.ts` following the skeleton from 21-RESEARCH.md (Pattern 1-5):

1. **Core loop:** Iterate commits by row index. For each commit:
   - If `commit.oid === '__wip__'`: create WIP node, find HEAD row via `is_head` scan, emit single dashed edge, `continue` (skip normal processing per Pitfall 2).
   - Otherwise: create `OverlayNode` with `x: commit.column, y: rowIndex, colorIndex: commit.color_index, isMerge: commit.is_merge, isBranchTip: commit.is_branch_tip, isStash: commit.is_stash, isWip: false`.

2. **Edge processing:** For each `edge` in `commit.edges`:
   - **Straight (from_column === to_column):** Check `activeLanes` Map for existing entry at this column. If same `colorIndex` AND same `dashed` → continue (coalesce). Otherwise → flush old entry via `flushLane()`, start new entry with `{ startY: rowIndex, colorIndex: edge.color_index, dashed: edge.dashed }`.
   - **Connection (from_column !== to_column):** Emit directly as `OverlayEdge` with `fromX: edge.from_column, fromY: rowIndex, toX: edge.to_column, toY: rowIndex, colorIndex: edge.color_index, dashed: edge.dashed`.

3. **Lane cleanup per row:** After processing all edges for a row, check each active lane. If a column has an active lane but NO straight edge in this row → flush that lane (it terminated). Delete from `activeLanes`.

4. **End flush:** After loop, flush ALL remaining `activeLanes` entries.

5. **`flushLane` helper:** Only emit edge if `endY > lane.startY` (avoid zero-length edges).

6. Return `{ nodes, edges, maxColumns }`.

**Key decisions matching research:**
- Connection edges get `fromY === toY === rowIndex` (path builder determines curve direction from fromX/toX delta)
- Read `edge.dashed` directly from backend data (Pitfall 3 — don't infer from `is_stash`)
- WIP handled with `continue` before normal edge processing (Pitfall 2)
- Coalesce only when column AND colorIndex AND dashed all match (Pitfall 4)
- Straight edges and connection edges are independent — a row can have both (Pitfall 6)
