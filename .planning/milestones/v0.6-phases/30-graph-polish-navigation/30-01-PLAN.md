In src-tauri/src/commands/branches.rs, inside the existing `#[cfg(test)]` module at the bottom of the file, add a new test `test_resolve_ref_inner`:

```rust
#[test]
fn test_resolve_ref_inner() {
    use git2::Signature;

    let work_dir = tempfile::TempDir::new().unwrap();
    let repo = git2::Repository::init(work_dir.path()).unwrap();
    let sig = Signature::now("Test", "test@example.com").unwrap();
    let tree_oid = {
        let mut index = repo.index().unwrap();
        index.write_tree().unwrap()
    };
    let tree = repo.find_tree(tree_oid).unwrap();
    let commit_oid = repo
        .commit(None, &sig, &sig, "init", &tree, &[])
        .unwrap();
    repo.branch("main", &repo.find_commit(commit_oid).unwrap(), false).unwrap();

    let path = work_dir.path().to_string_lossy().to_string();
    let mut state_map = std::collections::HashMap::new();
    state_map.insert(path.clone(), work_dir.path().to_path_buf());

    let result = resolve_ref_inner(&path, "main", &state_map);
    assert!(result.is_ok(), "resolve_ref_inner should succeed for valid branch");
    assert_eq!(
        result.unwrap(),
        commit_oid.to_string(),
        "resolved OID should match the branch tip commit"
    );

    let bad_result = resolve_ref_inner(&path, "refs/heads/nonexistent", &state_map);
    assert!(bad_result.is_err(), "resolve_ref_inner should fail for nonexistent ref");
}
```

This test will fail (RED) until `resolve_ref_inner` is implemented in Plan 30-02.
</action>

### Task 30-01-02: Allow graph column to shrink below lane content width (GRAPH-02)

<read_first>
- src/components/CommitRow.svelte (line 53 — graph column div with min-width)
- src/components/CommitGraph.svelte (lines 96-115 — startColumnResize, minWidths object)
</read_first>

<action>
**Step 1: Remove enforced min-width from CommitRow.svelte**

In CommitRow.svelte line 53, change:
```svelte
<div class="relative z-[1] flex items-center flex-shrink-0" style="width: {columnWidths.graph}px; min-width: {Math.max(maxColumns, commit.column + 1) * LANE_WIDTH}px;">
```
to:
```svelte
<div class="relative z-[1] flex items-center flex-shrink-0 overflow-hidden" style="width: {columnWidths.graph}px;">
```

Remove the `min-width` expression entirely. Add `overflow-hidden` so SVG content clips rather than overflows.

**Step 2: Lower the graph column minimum in CommitGraph.svelte**

In CommitGraph.svelte startColumnResize function (around lines 100-106), the minWidths object contains:
```typescript
graph: Math.max(maxColumns, 1) * displaySettings.laneWidth,
```
Change it to:
```typescript
graph: 20,
```

This allows the graph column to be resized down to 20px minimum.

**Step 3: Verify the header column also shrinks**

In CommitGraph.svelte template (around line 606), the graph header cell:
```svelte
<div class="flex-shrink-0 relative px-2" style="width: {columnWidths.graph}px;">
```
This already uses `columnWidths.graph` without a min-width — it will naturally shrink with the column. No change needed unless a min-width is found.
</action>
