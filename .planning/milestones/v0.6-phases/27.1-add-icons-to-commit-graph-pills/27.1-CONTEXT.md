# Phase 27.1: Add Icons to Commit Graph Pills - Context

**Gathered:** 2026-03-15
**Status:** Ready for planning

<domain>
## Phase Boundary

Add Lucide icons to all SVG ref pills in the commit graph overlay, replacing hand-drawn SVG path icons (tag diamond, stash flag) and adding icons to pill types that currently have none (local/remote branches). The visual language should match the icon choices already established in the sidebar and toolbar.

</domain>

<decisions>
## Implementation Decisions

### Icon choices per pill type
- **Local branch**: Laptop icon — matches sidebar BranchRow convention
- **Remote branch**: Globe icon — matches sidebar BranchRow convention
- **Tag**: Lucide Tag icon — replaces hand-drawn diamond, matches sidebar
- **Stash**: Archive icon — replaces hand-drawn flag, matches toolbar Stash button
- Consistent visual language: graph pills use the same icons as their sidebar/toolbar counterparts

### Claude's Discretion
- HEAD pill visual distinction (beyond existing bold text + full brightness)
- Rendering approach (foreignObject with Lucide components vs raw SVG path data injection)
- Icon sizing within the current ICON_WIDTH (10px) constraint — may need adjustment
- Whether hover expansion pills (showing all refs) also render icons
- How to handle `hasIcon()` function expansion (currently only returns true for Tag/Stash)

</decisions>

<specifics>
## Specific Ideas

- User consistently chose "match sidebar" / "match toolbar" for all icon selections — the guiding principle is visual consistency across the entire app, not graph-specific icons
- All four pill types (local branch, remote branch, tag, stash) should have icons — no pill type should be icon-less

</specifics>
