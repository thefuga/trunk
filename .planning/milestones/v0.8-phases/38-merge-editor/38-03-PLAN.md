    Create src/components/MergeEditor.svelte with the following structure:

    **Props interface:**
    ```typescript
    interface Props {
      repoPath: string;
      filePath: string;
      onclose: () => void;
      onresolved: () => void;
    }
    ```

    **Imports:**
    - `safeInvoke, type TrunkError` from '../lib/invoke.js'
    - `showToast` from '../lib/toast.svelte.js'
    - `type MergeSides` from '../lib/types.js'
    - `parseConflictRegions, computeOutput, takeAllCurrent, takeAllIncoming, toggleHunk, toggleLine, getConflictIndices, type ConflictRegion` from '../lib/merge-parser.js'
    - `Check, CircleCheck, CircleX, ChevronUp, ChevronDown` from '@lucide/svelte'

    **State (Svelte 5 $state runes):**
    - `regions = $state<ConflictRegion[]>([])`
    - `takenLines = $state<Set<string>>(new Set())`
    - `manualEdit = $state(false)`
    - `manualText = $state('')`
    - `loading = $state(true)`
    - `error = $state<string | null>(null)`
    - `focusedConflictIdx = $state(0)`
    - `panelRefs: HTMLDivElement[] = []` (3 elements for current, incoming, output scroll containers)

    **Derived state ($derived):**
    - `conflictIndices = $derived(getConflictIndices(regions))`
    - `outputText = $derived.by(() => { if (manualEdit) return manualText; return computeOutput(regions, takenLines); })`
    - `hasPrev = $derived(focusedConflictIdx > 0)`
    - `hasNext = $derived(focusedConflictIdx < conflictIndices.length - 1)`
    - `hasConflicts = $derived(conflictIndices.length > 0)`

    **$effect for data loading:**
    On mount (and when filePath changes), call `safeInvoke<MergeSides>('get_merge_sides', { path: repoPath, filePath })`. On success, call parseConflictRegions(result.base, result.ours, result.theirs), set regions, clear takenLines/manualEdit, set loading=false. On error, set error message, loading=false.

    **Synchronized scroll handler:**
    ```typescript
    let scrolling = false;
    function handleScroll(sourceIdx: number) {
      if (scrolling) return;
      scrolling = true;
      const scrollTop = panelRefs[sourceIdx].scrollTop;
      panelRefs.forEach((el, i) => {
        if (el && i !== sourceIdx) el.scrollTop = scrollTop;
      });
      requestAnimationFrame(() => { scrolling = false; });
    }
    ```

    **Event handlers:**
    - `handleTakeAllCurrent()`: `takenLines = takeAllCurrent(regions); manualEdit = false;`
    - `handleTakeAllIncoming()`: `takenLines = takeAllIncoming(regions); manualEdit = false;`
    - `handleToggleHunk(side: 'ours' | 'theirs', regionIdx: number)`: `takenLines = toggleHunk(side, regionIdx, regions, takenLines); manualEdit = false;`
    - `handleToggleLine(key: string)`: `takenLines = toggleLine(key, takenLines); manualEdit = false;`
    - `handleOutputEdit(e: Event)`: Set `manualEdit = true; manualText = (e.target as HTMLTextAreaElement).value;`
    - `handlePrevConflict()`: If hasPrev, decrement focusedConflictIdx, scroll the conflict region into view in all panels
    - `handleNextConflict()`: If hasNext, increment focusedConflictIdx, scroll the conflict region into view in all panels
    - `handleSaveAndResolve()`: Call `safeInvoke('save_merge_result', { path: repoPath, filePath, content: outputText })`. On success, `showToast('Resolved ' + filePath, 'success')` and call `onresolved()`. On error, show toast error.

    **Template layout (CSS grid/flex):**

    Outer container: `height: 100%; display: flex; flex-direction: column;`

    Top row (50% height): `flex: 1; display: flex; min-height: 0;`
    - Left panel (Current/Ours, 50% width): `flex: 1; display: flex; flex-direction: column; min-width: 0; border-right: 1px solid var(--color-border);`
      - Header bar: `height: 28px; background: var(--color-merge-current-header); border-bottom: 1px solid var(--color-merge-current-border); display: flex; align-items: center; padding: 0 8px; gap: 8px;`
        - Label: "Current (Ours)" in font-size 12px, color var(--color-text)
        - "Take All Current" button (styled like existing hunk action buttons)
      - Scrollable content: `flex: 1; overflow-y: auto; font-family: var(--font-mono); font-size: 12px; line-height: 18px;` bound to `panelRefs[0]`, `onscroll={() => handleScroll(0)}`
        - For each region: if context, render oursLines with neutral background. If conflict, render a hunk header row (clickable, calls handleToggleHunk('ours', regionIdx)), then each oursLine as a row with:
          - Line number gutter (width: 48px, color var(--color-text-muted), text-align right, padding-right 8px)
          - Check icon gutter (width: 20px): if line is taken, show Check icon (color var(--color-merge-taken-check), on hover show CircleX in var(--color-merge-remove-icon)). If not taken, show nothing (on hover show CircleCheck dimmed).
          - Line content (padding-left 4px)
          - Background: if conflict region, use var(--color-diff-add-bg) for ours lines. If taken, full opacity. If not taken, opacity var(--color-merge-dimmed).
          - Click handler: `handleToggleLine('ours-${regionIdx}-${lineIdx}')`

    - Right panel (Incoming/Theirs, 50% width): Same structure as left but:
      - Header background: var(--color-merge-incoming-header), border: var(--color-merge-incoming-border)
      - Label: "Incoming (Theirs)"
      - "Take All Incoming" button
      - Lines use theirsLines, keys use `theirs-${regionIdx}-${lineIdx}`
      - Conflict background: var(--color-diff-delete-bg) for theirs lines (they are "incoming changes")
      - Scroll: `panelRefs[1]`, `onscroll={() => handleScroll(1)}`

    Bottom panel (Output, 50% height): `flex: 1; display: flex; flex-direction: column; min-height: 0; border-top: 1px solid var(--color-border);`
    - Header bar: `height: 28px; background: var(--color-merge-output-header); border-bottom: 1px solid var(--color-merge-output-border); display: flex; align-items: center; padding: 0 8px; gap: 8px;`
      - Label: "Output" in font-size 12px
      - Spacer flex-1
      - Prev conflict button (ChevronUp icon, disabled when !hasPrev, color var(--color-text-muted))
      - Conflict counter: `{focusedConflictIdx + 1}/{conflictIndices.length}` in font-size 11px
      - Next conflict button (ChevronDown icon, disabled when !hasNext)
      - "Save and Mark Resolved" button (styled like the continue button: var(--color-btn-continue), var(--color-btn-continue-bg), var(--color-btn-continue-border))
      - Close button (X icon, calls onclose)
    - Textarea: `flex: 1; width: 100%; resize: none; border: none; background: var(--color-bg); color: var(--color-text); font-family: var(--font-mono); font-size: 12px; line-height: 18px; padding: 4px 8px; outline: none;`
      - `value={outputText}`
      - `oninput={handleOutputEdit}`
      - This is the editable output panel (CONF-06)

    **Loading/error states:**
    - While loading, show centered spinner or "Loading..." text
    - On error, show error message with retry button

    **Conflict navigation scroll-into-view:**
    When focusedConflictIdx changes, find the DOM element for that conflict region (use data-region-idx attribute) and call scrollIntoView({ behavior: 'smooth', block: 'center' }) on it in all three panels.

    **Hunk header row design:**
    Each conflict region starts with a separator row styled as:
    - Full width, height 24px, background var(--color-surface), border-top and border-bottom 1px solid var(--color-border)
    - Text: "Conflict {n}" where n is the conflict number (1-indexed among conflict regions only)
    - Cursor: pointer
    - Left side: check/empty icon indicating if all lines in this hunk are taken
    - Clicking the row calls handleToggleHunk for the appropriate side
