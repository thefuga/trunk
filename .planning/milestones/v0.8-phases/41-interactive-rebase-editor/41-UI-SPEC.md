## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, column padding (`COLUMN_PADDING_X`), inline padding, toolbar button vertical padding, action dropdown padding |
| sm | 8px | Row internal padding, button gaps, toolbar element spacing |
| md | 16px | Section padding (dialog internal padding) |
| lg | 24px | Column header height, toolbar height |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: 26px row height is inherited from the CommitGraph `ROW_HEIGHT` constant (`src/lib/graph-constants.ts` line 4). This value is a legacy system constraint shared across CommitGraph, CommitRow, overlay path calculations, and ref pill positioning. It cannot be changed without breaking layout consistency across the entire graph rendering pipeline. The rebase editor must use this same value to maintain visual parity with CommitGraph rows.

## Typography

| Role | Size | Weight | Line Height | Font Family |
|------|------|--------|-------------|-------------|
| Column header | 11px | 400 | 24px (container height) | `var(--font-sans)` |
| Row text (SHA, message, author, date) | 13px | 400 | 26px (row height) | `var(--font-sans)` |
| SHA column | 13px | 400 | 26px | `var(--font-mono)` |
| Action dropdown text | 11px | 400 | normal | `var(--font-sans)` |
| Toolbar button text | 11px | 600 | normal | `var(--font-sans)` |
| Validation error text | 11px | 400 | 1.3 | `var(--font-sans)` |
| Dialog title (InputDialog) | 14px (text-sm) | 600 | 1.4 | `var(--font-sans)` |
| Dialog body/textarea | 14px (text-sm) | 400 | 1.5 | `var(--font-sans)` |
| Dialog label | 12px (text-xs) | 400 | 1.4 | `var(--font-sans)` |

Typography rationale: The 11px/12px/13px/14px scale is intentional for information-dense developer tool UI. 11px serves column headers, toolbar buttons, dropdowns, and validation text (tertiary chrome). 12px serves dialog field labels (secondary chrome within modal context). 13px serves primary row content (the data the user is actively reading/editing). 14px serves dialog title and body (modal focus context where readability at slightly larger size is warranted). Each size corresponds to a distinct information hierarchy tier.

## Component Inventory

### RebaseEditor (center pane replacement)

**Layout:** Full-height flex column. Three sections top-to-bottom:
1. **Toolbar** (flex-shrink: 0, 36px height) — title text + action buttons
2. **Column header** (flex-shrink: 0, 24px height) — mirrors CommitGraph header
3. **Commit list** (flex: 1, overflow-y: auto) — scrollable row list

**Toolbar layout:** `display: flex; align-items: center; gap: 8px; padding: 0 12px;`
- Left: Title text "Interactive Rebase" (13px, weight 600, `--color-text`)
- Left (after title): Commit count badge "N commits" (11px, `--color-text-muted`)
- Right: Reset button, Discard Changes button, Start Rebase button (flex-shrink: 0, gap: 8px)

**Column header:** Identical structure to CommitGraph header. Height 24px. Background `var(--color-surface)`. Border-bottom 1px solid `var(--color-border)`. Font 11px `var(--color-text-muted)`.

Columns (left to right):

| Column | Width | Resizable | Hideable | Notes |
|--------|-------|-----------|----------|-------|
| Action | 90px fixed | No | No (always visible) | Dropdown select |
| SHA | 80px default | Yes (min 50, max 120) | Yes | Monospace font |
| Message | flex-1 | N/A (fills remaining) | No (always visible) | Text overflow: ellipsis |
| Author | 120px default | Yes (min 60, max 400) | Yes | Text overflow: ellipsis |
| Date | 100px default | Yes (min 60, max 400) | Yes | Relative format (Nm, Nh, Nd ago) |

Column widths and visibility persisted via LazyStore under separate keys from CommitGraph (key prefix: `rebase_column_widths`, `rebase_column_visibility`).

### Commit Row

**Height:** 26px (matching `ROW_HEIGHT` constant from CommitGraph — see Spacing Scale exceptions for rationale).
**Layout:** `display: flex; align-items: center; font-size: 13px; color: var(--color-text);`
**Cursor:** `grab` on hover (entire row is draggable). `grabbing` while dragging.
**Focus indicator:** 2px left border in `var(--color-accent)` on focused row. Background `var(--color-selected-row)`.
**Hover:** Background `var(--color-surface)` (same as CommitGraph rows).

**Drop state:**
- Opacity: `var(--color-rebase-drop-opacity)` (0.4)
- Text decoration: `line-through` on Message, Author, Date columns
- Action dropdown remains fully opaque and interactive

**Drag state (active drag):**
- Dragged row: `opacity: 0.6; pointer-events: none;`
- Drop target indicator: 2px solid `var(--color-accent)` border-top on the row being hovered over

### Action Dropdown (per row)

**Element:** Native `<select>` element (not custom dropdown).
**Styling:**
- Background: `var(--color-bg)`
- Border: 1px solid `var(--color-border)`
- Color: `var(--color-text)`
- Font size: 11px
- Padding: 4px 4px
- Border-radius: 3px
- Cursor: pointer
- Width: auto (fits content)

**Color dot indicator:** 6px diameter circle before the selected action text, colored per the Action Color Mapping table above. Implemented as a small inline span with `border-radius: 50%; width: 6px; height: 6px; background: var(--color-rebase-{action})`.

### Validation Error (inline)

**Placement:** Below the problematic row, spanning the full row width.
**Styling:**
- Background: `var(--color-rebase-error-bg)`
- Padding: 4px 12px (aligns with row content area)
- Font size: 11px
- Color: `var(--color-rebase-error)`
- No border (background tint is sufficient)

### Toolbar Buttons

Three buttons, right-aligned in toolbar:

| Button | Style | Enabled State | Disabled State |
|--------|-------|---------------|----------------|
| Reset | Ghost: `bg: var(--color-bg); border: 1px solid var(--color-border); color: var(--color-text)` | Normal | `opacity: 0.5` when commits match original state |
| Discard Changes | Ghost: `bg: var(--color-bg); border: 1px solid var(--color-border); color: var(--color-text)` | Always enabled | N/A |
| Start Rebase | Primary: `bg: var(--color-accent); color: white; font-weight: 600` | Normal | `opacity: 0.5` when validation errors exist |

All buttons: `border-radius: 4px; padding: 4px 12px; font-size: 11px; cursor: pointer; white-space: nowrap;`

### InputDialog (reword/squash message editor)

Reuse existing `InputDialog.svelte` component with new `confirmLabel` and `cancelLabel` props. The component currently hardcodes "Cancel" and "OK" as button labels. These must become configurable props with the current text as defaults:

**New props required:**
- `confirmLabel?: string` (default: "OK")
- `cancelLabel?: string` (default: "Cancel")

**Existing features (unchanged):**
- Modal overlay with backdrop (rgba(0,0,0,0.5))
- Surface card with 16px padding, rounded-lg, shadow-xl
- Title (text-sm, font-semibold)
- Multiline textarea (text-sm, themed bg/border/color)
- Escape to cancel, auto-focus first field

**Reword dialog configuration:**
- title: "Reword Commit"
- confirmLabel: "Save Message"
- cancelLabel: "Keep Original"
- fields: single multiline field, key "message", label "Commit Message", prefilled with existing commit message

**Squash dialog configuration:**
- title: "Squash Commits"
- confirmLabel: "Save Message"
- cancelLabel: "Keep Original"
- fields: single multiline field, key "message", label "Commit Message", prefilled with concatenated messages from all squashed commits

## Interaction Contract

### Keyboard Shortcuts

All shortcuts active ONLY when a rebase editor row is focused (not when InputDialog or other inputs have focus):

| Key | Action | Condition |
|-----|--------|-----------|
| P | Set focused row action to Pick | Row focused, not in text input |
| S | Set focused row action to Squash | Row focused, not in text input |
| R | Set focused row action to Reword | Row focused, not in text input |
| D | Set focused row action to Drop | Row focused, not in text input |
| ArrowUp | Move focus to previous row | Row focused |
| ArrowDown | Move focus to next row | Row focused |
| Escape | Close editor (discard changes) | Editor has focus, no dialog open |

### Drag-and-Drop Reorder

- **Mechanism:** Native HTML5 Drag and Drop API
- **Trigger:** `draggable="true"` on each row div
- **Visual feedback:** Dragged row at reduced opacity (0.6). Target position indicated by 2px accent border-top on the row under the cursor.
- **Swap behavior:** Rows swap positions in real-time during dragover (not just on drop). The array reorders on each dragover event.
- **Animation:** CSS transition on `transform` property for rows shifting up/down during drag reorder. Duration: 150ms ease.

### Focus Management

- Clicking a row sets it as focused (keyboard shortcut target)
- Tab into the editor focuses the first row
- ArrowUp/ArrowDown navigate between rows
- Only one row focused at a time
- Focus state: left 2px accent border + `--color-selected-row` background

### Validation

Validation runs on every action change and every reorder. Errors display inline below the problematic row.

| Rule | Error Message | Placement |
|------|---------------|-----------|
| First non-dropped commit is squash | "Cannot squash the first commit" | Below the offending row |
| All commits set to drop | "Cannot drop all commits" | Below the first row |
| Squash with no non-dropped predecessor | "No preceding commit to squash into" | Below the offending row |

"Start Rebase" button disabled (opacity 0.5, cursor default) when any validation error exists.

## State Transitions

```
[CommitGraph] --right-click commit--> "Interactive Rebase..." --> [RebaseEditor]
[CommitGraph] --right-click branch--> "Interactive Rebase {branch}..." --> [RebaseEditor]

[RebaseEditor] --Discard Changes--> [CommitGraph]
[RebaseEditor] --Start Rebase (valid)--> [CommitGraph + OperationBanner]
[RebaseEditor] --Start Rebase (invalid)--> [RebaseEditor with inline errors]

[CommitGraph + OperationBanner] --reword/squash pause--> [InputDialog over CommitGraph]
[InputDialog] --"Save Message"--> [CommitGraph + OperationBanner continues]
[InputDialog] --"Keep Original"--> abort message edit (git fails, shows error toast)

[CommitGraph + OperationBanner] --conflict--> [MergeEditor via Phase 37-38]
[CommitGraph + OperationBanner] --complete--> [CommitGraph (graph refreshes, no toast)]
```
